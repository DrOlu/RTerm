# v2.1.0 Release Notes

## English

v2.1.0 adds four high-value capability areas on top of the v2.0.0 SRE pillar: **APM (application performance monitoring consumer)**, **Infrastructure monitoring for Kubernetes/cloud**, **Digital Experience Monitoring (RUM)**, and **agentless ETW-based Windows diagnostics** — all fully unit-tested (26 new tests, 1001 total) and built as pure, injectable modules that compose with the existing SRE ledgers, watchdogs, and MOP.

### APM — distributed-trace consumer (`apm/spanLedger.ts`)

RTerm now ingests and analyzes OpenTelemetry spans (OTLP), answering code-level questions without instrumenting app code itself (that's the OTel SDK's job). The `SpanLedger` stores spans per trace and provides:
- **Trace summaries** — root service, span count, total duration, error presence.
- **Per-service stats** — span count, error rate, duration p50/p95/p99/max.
- **Slowest traces** and **bottleneck services** (ranked by error count).
- **OTLP/HTTP JSON parsing** (`parseOtlpJson` / `ingestOtlp`) — point your apps' OTLP exporter at RTerm and it stores + analyzes the spans.

### Infrastructure monitoring for Kubernetes/cloud (`infra/infraMonitor.ts`)

Kubernetes/container and cloud-service health, normalized for the existing SRE ledgers and watchdogs:
- **`parseKubectlPods`** — parse `kubectl get pods` output into structured pods (phase, restarts, ready).
- **`clusterHealth`** — running/not-ready/CrashLoopBackOff pods, total restarts, node readiness, and **CPU/mem % of limit**.
- **`podUnhealthy`** + `unhealthyInstances` — flag pods/instances needing attention for alerting.

### Digital Experience Monitoring (`dem/rumLedger.ts`)

Real User Monitoring: a tiny JS beacon on real pages POSTs Core Web Vitals; RTerm stores sessions and answers UX questions per page/route/region:
- **`ingestBeacon`** — accept the beacon payload (LCP, INP, CLS, TTFB, JS errors, region).
- **Per-page stats** — sessions, **p75 LCP/INP/TTFB**, avg CLS, **error rate**.
- **Slowest/poor pages** (over the Core Web Vitals LCP threshold) — pairs with the existing SyntheticChecks (the synthetic half of DEM).

### ETW-based Windows diagnostics (`etw/etwService.ts`)

Agentless Windows diagnostics using **built-in ETW tooling** (no install, no AV risk — the better alternative to third-party tracers):
- **Built-in provider catalog** — network, file, registry, process, DNS, power (Microsoft-Windows-Kernel-*).
- **`buildStartCommands`/`buildStopCommands`** — logman trace-session commands (network/file/registry providers).
- **`buildWinEventQuery`/`buildCounterQuery`** — `Get-WinEvent`/`Get-Counter` JSON queries.
- **`parseWinEventJson`/`parseWinEventText`/`parseCounterJson`** — parse the output into structured events with **field extraction** (ip, port, path, registry key, process name, pid), and **`topCounters`** for top-consuming processes.

All four areas are **pure + injectable** (execution/fetch injected — WinRM `executeCommand` for ETW, kubectl/aws/az for infra, beacon POST for RUM, OTLP exporter for APM), following RTerm's established pattern, and feed the existing `metricsLedger`, `uptimeWatchdog`, `goldenSignals`, `alertService`, `incidentLedger`, and MOP.
