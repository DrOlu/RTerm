# v2.4.0 Release Notes

## English

v2.4.0 adds **dagu workflow support** to RTerm: run declarative [dagu](https://github.com/dagucloud/dagu) YAML DAG workflows natively on RTerm's orchestrated playbook engine ‚Äî no dagu server required.

### dagu workflow support (`dagu/daguParser.ts`)

A parser that compiles dagu YAML workflows into RTerm playbooks, which already support `dependsOn` DAGs, failure policies, validation, rollback, runbook params, and retry. dagu's declarative DAG format maps cleanly onto RTerm's orchestrated runner:

- **Steps** ‚Äî `id`/`name` ‚Üí step id/name; `run`/`command`/`cmd`/`script`/`call` ‚Üí the step command (all forms supported).
- **Dependencies** ‚Äî `depends` (string or array) ‚Üí `dependsOn` (fan-out/fan-in DAGs, executed in topological waves by `dagScheduler`).
- **Failure handling** ‚Äî `continue_on` ‚Üí `onError: continue`; `retry_policy` ‚Üí noted (RTerm retries via the runbook engine).
- **Guards** ‚Äî `preconditions` ‚Üí a `desiredState` skip-when-satisfied guard.
- **Runbook params** ‚Äî `params` (string or object with `default`) ‚Üí playbook params with defaults.
- **Metadata** ‚Äî `name`, `description` carried through; `maxParallelSteps` set for DAG execution.
- **Safety** ‚Äî `validateDag` runs up front (cycles / missing dependencies surfaced as warnings), and `daguExecutionPlan` summarizes the execution waves.

### Run a dagu workflow

Paste dagu YAML to the agent ("run this dagu workflow") or compile it via `parseDaguYaml` and run the resulting playbook with `run_playbook` ‚Äî it executes on RTerm's orchestrated DAG runner across your hosts with validation and rollback.
