# v1.6.0 Release Notes

## English

v1.6.0 is a workspace navigation and long-session clarity release. The headline addition is the **global Tab List panel**: GyShell can now show terminal tabs and chat sessions as a vertical inventory, independent from the panels currently hosting those tabs. This makes large workspaces easier to scan, reopen, drag, close, and extend. The other major change is persistent **compaction boundary markers**: when GyShell compacts old chat history, the UI now records and renders a `[CTX COMPACTED]` separator at the actual retained-history cutoff across desktop, mobile-web, and TUI clients. This release also hardens deterministic compaction fallback recovery, provider-stream handling, terminal title uniqueness, detached-window terminal bootstrapping, and background terminal creation from the new list panel.

### Highlights You Will Notice

- **Global Tab List panel**
  - a new `TAB LIST` panel is available from the workspace rail and is included in the default main layout on the left, with chat in the center and terminal on the right
  - the panel has Terminal and Chat modes, each with counts, icons, subtitles, status dots, and newest-first ordering
  - terminal rows show local/SSH state, while chat rows show message counts and only use the running indicator for busy sessions
  - selecting a hosted row activates the panel that already owns that tab; selecting an unhosted inventory row restores it into the primary panel for that kind
  - rows expose the same tab drag payloads as normal panel headers, so terminal and chat tabs can still be moved through existing drag/drop and detached-window flows
  - the list panel can create new chat sessions, local terminals, and saved-SSH terminals; local and SSH terminal tabs created from the list panel can start in the background without forcing a terminal panel to appear
  - terminal closes requested from the list panel go through the existing terminal close confirmation path
- **More predictable background terminal creation**
  - background local and SSH terminal tabs remain visible in the global terminal inventory and stay bindable to terminal panels
  - local terminal tabs created from the list panel now start their backend runtime even when no terminal panel exists, but they stay unhosted until a terminal panel is available
  - if a terminal panel already exists, it can host a newly created background terminal tab immediately; if no terminal panel exists, a future terminal panel can still pick it up automatically
  - linked filesystem and monitor panels stay hidden from background SSH tabs until the tab is explicitly opened for those panel kinds
  - detached windows can materialize transferred SSH terminal placeholders before the backend runtime has appeared, preserving the SSH config needed to start the terminal later
- **Persistent compaction boundary markers**
  - the agent now emits a `compaction_boundary` UI event at the real cutoff where old history was summarized and the protected tail begins
  - desktop chat renders that point as a compact `[CTX COMPACTED]` separator, instead of letting long-session history maintenance remain invisible
  - the marker is stored in UI history, survives SQLite flush/reload, and is de-duplicated when duplicate boundary events arrive
  - rollback and branch operations normalize markers so boundaries whose anchors were removed do not linger in the wrong place
  - mobile-web and TUI clients now understand `INSERT_MESSAGE` updates and render or summarize the same boundary marker consistently
  - previews and auto-title logic skip compaction boundary rows, and the marker does not steal assistant turn copy/branch controls in classic or Seamless chat display
- **Deterministic fallback compaction recovery**
  - if the compaction model fails or returns an empty summary, GyShell now builds a local deterministic digest instead of simply skipping compaction
  - fallback summaries keep the protected tail exact, include the compaction failure reason, and stay under a hard character cap even with huge provider diagnostics
  - when the UI anchor is available, GyShell exports the older pre-tail UI history to a managed Markdown file and includes Pass Chat-style local-read guidance so exact old details can still be inspected on demand
  - fallback compaction respects previous compaction summaries and pruned tool materialization, so raw material hidden before an earlier compaction is not reintroduced
  - aborts do not trigger fallback compaction; if cancellation happens after an export is written, the unreferenced export is cleaned up instead of leaving stale files
  - deleting sessions, rolling back past a fallback summary, and branching from fallback-compacted sessions now preserve or remove managed history exports according to ownership and remaining references
- **Empty provider stream finishes retry instead of ending silently**
  - empty, non-tool model stream finishes with provider finish reasons such as `error` are now treated as unusable responses and routed through the normal retry path
  - valid tool-call finishes remain routable even when assistant text is empty
  - transient empty provider-error chunks are not persisted into backend chat history, and the UI receives a visible retry alert before recovery
- **Terminal title and inventory stability**
  - explicit user titles with numeric suffixes are preserved instead of being reinterpreted as auto-generated suffixes
  - backend and renderer inventories keep display titles unique and stable across duplicate backend snapshots and concurrent terminal spawns
  - transferred terminal placeholders also receive unique display/config titles before backend reconciliation catches up
- **Packaged mobile-web runtime refreshed**
  - the Electron-packaged mobile-web runtime assets were regenerated so desktop builds serve the updated client code without requiring a separate mobile-web development server

### Under the Hood

- **List panel as a first-class panel kind**
  - `listPanel` is now part of the layout model, rail metadata, panel renderer registry, panel constraints, saved layout parsing, and default legacy layout projection
  - list rows are built from global tab inventory plus panel ownership state, so hosted, unhosted, suppressed, and cross-window rows are resolved before rendering
  - terminal creation UI was extracted into `TerminalAddButton` and reused by both terminal panels and the list panel
- **Layout and tab visibility plumbing**
  - layout binding now distinguishes owned tab inventory from layout-bindable tabs, allowing background SSH tabs to be visible globally while staying hidden from linked filesystem/monitor panels
  - list-panel local terminal creation can request backend runtime startup without creating a terminal panel, and newly created tabs only activate an existing host instead of forcing one into the layout
  - cross-window and detached-window bootstrap paths preserve terminal snapshots so transferred terminal tabs can appear before backend list events arrive
- **Compaction boundary persistence**
  - `AgentService_v2` assigns a stable backend id to the compaction summary message and emits boundary metadata for the target, previous anchor, summary id, and protected-round count
  - `UIHistoryService` inserts boundary messages before the target anchor or after the previous anchor, persists them, and normalizes them after rollback and branch operations
  - desktop `ChatStore`, mobile-web session state, and TUI state all share the same anchor-based insertion and normalization behavior
- **Deterministic compaction fallback internals**
  - `buildDeterministicCompactionDigest` selects leading system/first-user context plus recent digest entries, clips large entries from the middle, and records selected/omitted counts
  - fallback history exports use the Pass Chat export service in grouped-by-session mode under the history storage directory, with owner-only permissions where supported and no global max-file pruning
  - branch creation rewrites managed fallback export paths into branch-owned files, while unmanaged paths or paths owned by another session are left untouched
  - rollback and session deletion delete only managed fallback exports that are no longer referenced by the owning session
- **Stream-response validation**
  - streamed response helpers now distinguish malformed empty tool-call finishes from generic empty unusable responses and expose finish reasons for retry diagnostics
  - regression coverage verifies that empty provider-error streams retry, valid tool calls still execute, and transient empty responses do not pollute stored history
- **Tests**
  - new and expanded extreme specs cover list panel row modeling, default layout/list-panel registration, cross-window tab movement, background local/SSH creation and binding, unique terminal titles, detached SSH bootstrap, compaction boundary insertion/persistence/rollback, deterministic fallback compaction, managed fallback export lifecycle, branch export ownership, desktop/mobile/TUI boundary rendering, and empty stream retry behavior

### What This Means in Practice

- Workspaces with many terminal and chat tabs are easier to scan and reorganize because tab inventory is no longer tied only to the currently visible panel headers.
- Background local and SSH creation from the list panel behaves predictably: terminal panels can host those tabs, while filesystem and monitor panels do not suddenly change context.
- Long chats now show where old context was compacted, so the visible conversation has a clearer relationship to the agent's retained context.
- If the compaction model is unavailable or returns nothing, long sessions can still recover with a deterministic local digest plus an optional exact-history export for old details.
- Provider hiccups that produce empty stream endings are less likely to make a run appear finished without an answer.
- Terminal labels stay stable even when users intentionally use numeric suffixes or multiple terminals race to start.

## 中文

v1.6.0 是一次聚焦于工作区导航和长会话可读性的更新。最主要的新增是 **全局 Tab List 面板**：GyShell 现在可以把 terminal tab 和 chat session 作为垂直清单展示出来，而不是只能依赖当前可见 panel header 上的标签。大型工作区会更容易扫描、重新打开、拖动、关闭和扩展。另一个重要变化是持久化的 **上下文压缩边界标记**：当 GyShell 压缩旧聊天历史时，界面会在真实的保留历史切点记录并渲染 `[CTX COMPACTED]` 分隔线，而且桌面端、mobile-web 和 TUI 都会保持一致。本次更新还加强了确定性压缩兜底恢复、模型空流处理、终端标题唯一性、 detached window 终端启动占位，以及从新列表面板创建后台 terminal tab 时的绑定行为。

### 你会直接感知到的升级

- **全局 Tab List 面板**
  - 工作区 rail 中新增 `TAB LIST` 面板，并进入默认主布局：左侧是列表，中间是聊天，右侧是终端
  - 面板支持 Terminal / Chat 两种模式，展示数量、图标、副标题、状态点，并按最近更新时间排序
  - terminal 行会展示本地/SSH 状态；chat 行会展示消息数量，并且只有忙碌中的会话才显示运行中状态点
  - 点击已经被某个 panel 托管的行，会激活对应 panel 中的 tab；点击未托管的清单行，会把它恢复到该类型的主 panel
  - 行项目复用现有 tab 拖拽 payload，因此 terminal 和 chat tab 仍然可以走原来的拖拽、跨 panel 和 detached-window 流程
  - 列表面板可以创建新的 chat session、本地 terminal，以及已保存的 SSH terminal；从列表创建的本地和 SSH terminal tab 都可以在后台启动，不会强行打开 terminal panel
  - 从列表面板关闭 terminal 时，会走已有的 terminal 关闭确认流程
- **更可预测的后台 terminal 创建**
  - 后台本地和 SSH terminal tab 会继续出现在全局 terminal 清单中，并保持可绑定到 terminal panel
  - 从列表面板创建的本地 terminal tab 现在即使没有 terminal panel，也会启动后端 runtime，但会保持未托管状态直到 terminal panel 可用
  - 如果当前已有 terminal panel，新建的后台 terminal tab 可以立即被该 panel 托管；如果当前没有 terminal panel，未来恢复 terminal panel 时也能自动接住它
  - 关联的文件系统和监控 panel 不会因为后台 SSH tab 被创建而突然切换上下文，除非用户明确把该 tab 打开到这些 panel 类型
  - detached window 可以在后端 runtime 出现之前先物化被转移的 SSH terminal 占位，并保留之后启动所需的 SSH 配置
- **持久化的上下文压缩边界标记**
  - Agent 现在会在旧历史被总结、受保护尾部开始的真实位置发出 `compaction_boundary` UI 事件
  - 桌面聊天界面会把这个位置渲染成紧凑的 `[CTX COMPACTED]` 分隔线，让长会话历史维护不再是不可见的后台动作
  - 该标记会写入 UI history，能够经过 SQLite flush/reload 保留下来，并且重复边界事件不会产生重复标记
  - rollback 和 branch 会重新规范化边界标记；如果锚点消息已经被裁掉，边界不会错误地留在别的位置
  - mobile-web 与 TUI 现在都理解 `INSERT_MESSAGE` 更新，并能一致地渲染或摘要同一个边界标记
  - 预览和自动标题逻辑会跳过压缩边界行；在传统模式和 Seamless 模式中，边界也不会影响助手轮次的复制/分支控制
- **确定性压缩兜底恢复**
  - 如果 compaction 模型失败或返回空摘要，GyShell 现在会构建本地确定性 digest，而不是直接跳过压缩
  - fallback 摘要会保留受保护尾部的精确原文，包含压缩失败原因，并且即使 provider diagnostics 很大也会被限制在硬字符上限内
  - 如果 UI 锚点可用，GyShell 会把受保护尾部之前的旧 UI history 导出为受管理 Markdown 文件，并附上类似 Pass Chat 的本地读取指引，让旧细节仍可按需精确查看
  - fallback compaction 会尊重此前的压缩摘要和已裁剪工具内容，不会把更早压缩前隐藏的原始内容重新引入
  - abort 不会触发 fallback compaction；如果取消发生在导出文件写入之后，未被引用的导出文件也会被清理
  - 删除会话、rollback 越过 fallback 摘要、以及从 fallback 压缩后的会话 branch 时，现在会根据所有权和剩余引用正确保留或删除受管理历史导出
- **模型空流结束会重试，而不是静默结束**
  - 带有 `error` 等 provider finish reason 的空内容、非工具调用 stream 结束，现在会被视为不可用响应，并进入正常重试路径
  - 有效的工具调用结束即使没有助手文本，也仍然会被正常路由执行
  - 短暂的 provider 空错误不会写入后端聊天历史；UI 会先收到可见的重试提示，再展示恢复后的结果
- **终端标题与清单稳定性**
  - 用户显式设置的数字后缀标题会被保留，不会被误当成自动生成的 suffix 重新解释
  - 后端和渲染端清单在遇到重复 backend title 或并发创建 terminal 时，会保持唯一且稳定的显示标题
  - 跨窗口转移出来的 terminal 占位也会在后端 reconciliation 赶上之前获得唯一的显示标题和 config 标题
- **内置 mobile-web runtime 已刷新**
  - Electron 打包内置的 mobile-web 产物已重新生成，因此桌面构建直接托管的移动端页面也包含本次更新的客户端代码，不需要额外启动 mobile-web 开发服务

### 底层改动

- **List panel 成为一等 panel 类型**
  - `listPanel` 已接入 layout model、rail 元数据、panel renderer registry、panel 尺寸约束、已保存布局解析，以及默认 legacy layout 投影
  - 列表行由全局 tab inventory 和 panel ownership 状态共同生成，因此 hosted、unhosted、suppressed 和跨窗口行会在渲染前统一解析
  - terminal 创建按钮抽成 `TerminalAddButton`，供 terminal panel 和 list panel 复用
- **布局与 tab 可见性管线**
  - layout binding 现在区分“拥有的 tab 清单”和“可绑定到布局的 tab”，让后台 SSH tab 可以在全局可见，同时对关联的 filesystem/monitor panel 保持隐藏
  - list panel 创建本地 terminal 时可以请求后端 runtime 启动而不创建 terminal panel；新建 tab 只会激活已有 host，不会强行把 host 塞进布局
  - 跨窗口与 detached-window bootstrap 路径会保留 terminal snapshot，使被转移的 terminal tab 能在后端列表事件到达前先显示出来
- **压缩边界持久化**
  - `AgentService_v2` 会给 compaction summary message 分配稳定 backend id，并发出包含 target、previous anchor、summary id 和 protected-round count 的边界 metadata
  - `UIHistoryService` 会把边界消息插入到 target anchor 之前或 previous anchor 之后，持久化它，并在 rollback/branch 后重新规范化
  - 桌面 `ChatStore`、mobile-web session state 和 TUI state 都采用同一套基于 anchor 的插入与规范化行为
- **确定性压缩兜底内部机制**
  - `buildDeterministicCompactionDigest` 会选择前置 system / 首个用户上下文和较新的 digest entries，从中间裁剪超大条目，并记录选中/省略数量
  - fallback history export 复用 Pass Chat 导出服务的按 session 分组模式，落在 history storage 目录下；文件系统支持时使用 owner-only 权限，并且没有全局 max-file 清理
  - 创建 branch 时，受管理的 fallback export path 会被改写成 branch 自己拥有的文件；非受管理路径或属于其它 session 的路径保持不变
  - rollback 和 session 删除只会删除当前 session 拥有且已不再被引用的受管理 fallback export
- **流式响应校验**
  - streamed response helper 现在能区分“畸形空工具调用结束”和“通用空不可用响应”，并暴露 finish reason 供重试诊断使用
  - 回归测试覆盖空 provider-error stream 重试、有效工具调用继续执行，以及短暂空响应不污染持久化历史
- **测试**
  - 新增和扩展的 extreme 规格覆盖 list panel 行模型、默认布局/list-panel 注册、跨窗口 tab 移动、后台本地/SSH 创建与绑定、终端唯一标题、detached SSH bootstrap、压缩边界插入/持久化/回滚、确定性 fallback compaction、受管理 fallback export 生命周期、branch export 所有权、桌面/mobile/TUI 边界渲染，以及空流重试行为

### 这次升级带来的实际体验

- 当工作区里有很多 terminal 和 chat tab 时，你不再只能从当前可见 panel header 找 tab，整体扫描和重组都会更直接。
- 从列表面板创建后台本地或 SSH tab 的行为更稳定：terminal panel 可以接住这些 tab，但文件系统和监控 panel 不会突然被切换上下文。
- 长对话现在能明确显示旧上下文在哪里被压缩，当前可见聊天记录与 Agent 保留上下文之间的关系更清楚。
- 如果 compaction 模型不可用或返回空内容，长会话仍可通过本地确定性 digest 和可选的精确历史导出恢复。
- provider 偶发返回空 stream 时，运行更不容易像“已经完成但没有答案”一样静默结束。
- 即使用户主动使用数字后缀命名，或者多个 terminal 并发启动，终端标签也会保持稳定可读。
