# v1.4.3 Release Notes

## English

v1.4.3 makes the file workspace feel more like a real file manager and tightens up Seamless-mode and monitor panel polish: the built-in editor can now preview images and PDFs in place, paste-on-conflict gains a "Keep Both" option that auto-numbers instead of forcing you to overwrite, file rows get a "Copy Full Path(s)" entry in the right-click menu, Seamless tool groups are now visibly expandable and show the full call info on demand, and the compact horizontal monitor layout no longer collapses mid-value bars.

### Highlights You Will Notice

- **Image and PDF preview in the file editor**
  - the built-in editor can now open images directly — `png`, `jpg`, `gif`, `webp`, `bmp`, `ico`, `svg`, and `avif` — without sending you to an external viewer
  - PDFs are rendered inline through `pdfjs-dist`, with previous/next page navigation, zoom in/out, and a "page X / Y" indicator
  - the editor panel title is now "File Editor" (previously "Text Editor"), reflecting that it is no longer text-only
  - file preview detection for the editor is centralized: text, image, and PDF files are routed into the right preview mode automatically based on extension and size
  - Find (`Ctrl/Cmd+F`) is automatically closed and disabled while previewing non-text files, so the find shortcut doesn't open a search bar that has nothing to search
- **"Keep Both" when pasting into a folder with name conflicts**
  - the paste conflict dialog used to give you a single choice — overwrite or cancel — now you also get **Keep Both**
  - "Keep Both" copies/moves the conflicting items in with auto-generated numbered suffixes (e.g. `report.pdf` → `report (2).pdf`) so you keep the existing files untouched
  - conflict resolution is now driven by an end-to-end `conflictStrategy` (`error` / `overwrite` / `rename`), so the same semantics apply consistently across local and SSH file transfers
  - the conflict dialog message has been updated to reflect the new choice
- **Copy Full Path(s) from the file row right-click menu**
  - right-click on a file or folder row in the file panel and you'll see a new **Copy Full Path** entry (or **Copy Full Paths** when multiple items are selected)
  - the absolute path is written to your system clipboard, ready to paste anywhere — terminal, chat, an editor, etc.
  - a status message confirms how many paths were copied
- **More reliable SSH SFTP file writes**
  - the chunk-write API now closes the underlying SFTP handle on the final chunk (and on empty-file writes), so handles are released cleanly instead of being held open until session GC
  - in practice this makes large or repeated file uploads to SSH targets more reliable, especially around the end of a transfer
- **Seamless tool groups are visibly expandable**
  - the chevron next to a tool group header is now visible at rest (no longer only on hover), so you can tell at a glance that the group can be expanded
  - single-step groups now use the same expandable header as multi-step groups — no more inconsistency between "1 step" and "N steps" appearance
  - the `N steps` count now has proper spacing instead of hugging the title text
  - expanding a group reveals the **full, untruncated** call info for each step — full command, full tool arguments, full file path — which previously got clipped in the compact one-liner. Tool *output* is still not shown; this is about being able to inspect what was called, not what came back
- **Monitor panel: correct vertical bar heights in compact horizontal layout**
  - in the compact horizontal monitor layout, mid-value bars (such as DISK, LOAD, and RX) used to collapse to a thin sliver while CPU/MEM/TX bars rendered correctly
  - the bars now resolve their percentage heights reliably, so the visual reading matches the underlying numbers across all metrics
  - bars at 0% are now intentionally hidden instead of showing a minimum stub, making "no signal" visually distinct from "low signal"

### What This Means in Practice

- The file panel handles more of your day-to-day file workflow without leaving GyShell — open an image, flip through a PDF, copy a full path into a chat or terminal, all from the same panel.
- Pasting into a folder full of similarly-named files no longer forces a destructive choice: keep the originals and let GyShell number the new copies.
- Long-running file uploads over SSH finish more cleanly, with no lingering server-side handles after the last chunk.
- Seamless mode stops looking "static" — you can immediately tell which tool groups are expandable, and when you expand them you see the *complete* call info instead of a truncated preview.
- The compact monitor layout now visually reflects real usage; you no longer have to second-guess whether DISK or LOAD looks pinned just because of a layout bug.

## 中文

v1.4.3 让文件工作区更接近一个真正的文件管理器，并对 Seamless 模式和监控面板做了细致打磨：内置编辑器现在可以直接预览图片和 PDF；粘贴冲突时新增"保留两者"选项，自动添加数字后缀，不再被迫覆盖；文件右键菜单新增"复制完整路径"；Seamless 模式下工具组现在明显可展开，并支持查看完整调用信息；紧凑横向监控布局下的柱状条高度问题也已修复。

### 你会直接感知到的升级

- **文件编辑器支持图片和 PDF 预览**
  - 内置编辑器现在可以直接打开图片——`png`、`jpg`、`gif`、`webp`、`bmp`、`ico`、`svg`、`avif`——无需切换外部查看器
  - PDF 通过 `pdfjs-dist` 内联渲染，支持上一页 / 下一页翻页、放大 / 缩小、以及"第 X / Y 页"指示
  - 编辑器面板标题从"文本编辑器"更新为"文件编辑器"，反映它不再只能处理文本
  - 编辑器的文件预览能力检测被统一收口：文本、图片、PDF 会根据扩展名和大小自动路由到对应的预览模式
  - 在预览非文本文件时，`Ctrl/Cmd+F` 查找会自动关闭并禁用，避免在没有可搜索内容的页面上误触发搜索条
- **粘贴时新增"保留两者"选项**
  - 粘贴冲突对话框过去只能选择"覆盖"或"取消"，现在多了一个 **保留两者**
  - 选择"保留两者"后，冲突项会以自动编号后缀的形式复制 / 移动进来（例如 `report.pdf` → `report (2).pdf`），原有文件保持不变
  - 冲突处理改为由端到端的 `conflictStrategy`（`error` / `overwrite` / `rename`）驱动，本地和 SSH 文件传输共享一致的处理语义
  - 冲突对话框的提示文案已同步更新，明确说明新增的选项
- **文件行右键菜单新增"复制完整路径"**
  - 在文件面板中右键文件或文件夹行，可以看到新的 **复制完整路径** 菜单项（多选时显示为 **复制完整路径**，并复制多条）
  - 绝对路径会被写入系统剪贴板，可以直接粘贴到终端、对话、编辑器等任何地方
  - 操作完成后会有状态提示显示一共复制了多少条路径
- **SSH SFTP 写入更稳**
  - 分块写入接口在最后一个分块（以及空文件写入）时会主动关闭底层 SFTP 句柄，不再依赖会话 GC 才释放
  - 实际效果是：大文件或频繁上传到 SSH 目标时，传输尾段更稳定，不再容易留下悬挂的句柄
- **Seamless 模式工具组现在明显可展开**
  - 工具组标题右侧的箭头默认就有可见度，不再只有悬停时才显示，可以一眼看出"这一组是可以点开的"
  - 单步骤工具组现在也使用与多步骤组相同的可展开 header——不再出现"1 步"和"N 步"显示风格不一致的问题
  - `N steps` 步骤计数现在与标题之间有合适的间距，不再紧贴
  - 展开工具组后，每一步显示的是**完整、未截断**的调用信息——完整命令、完整工具参数、完整文件路径——以前在紧凑单行显示中会被裁掉。工具的*输出内容*仍然不会展示，这次仅是为了让你能查看"调用了什么"，而不是"返回了什么"
- **监控面板：紧凑横向布局下的柱状条高度修复**
  - 紧凑横向监控布局下，中间数值的柱状条（如 DISK、LOAD、RX）会塌缩成最低高度的细条，而 CPU / MEM / TX 等柱状条则正常显示
  - 现在所有柱状条都会按真实百分比稳定渲染，视觉读数与底层数据保持一致
  - 0% 的柱状条会刻意不显示填充块，让"没有信号"和"信号很弱"在视觉上能够明确区分

### 这次升级带来的实际体验

- 文件面板可以承担你日常更多的文件操作——打开图片、翻 PDF、把完整路径复制进对话或终端，都不需要离开 GyShell。
- 把内容粘贴到包含同名文件的目录时，不再被强迫做"覆盖或放弃"的选择：可以保留原文件，让 GyShell 给新副本自动编号。
- 通过 SSH 上传大文件，传输尾段更干净，不会再在服务端留下未释放的句柄。
- Seamless 模式不再显得"静态"——你可以马上看出哪些工具组可以展开，展开之后看到的是*完整*调用信息，而不是被截断的预览。
- 紧凑监控布局下，柱状条现在能真实反映使用率；不再需要怀疑"DISK 或 LOAD 是不是看起来满了，其实只是布局 bug"。
