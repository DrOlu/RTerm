# v1.4.2 Release Notes

## English

v1.4.2 tightens the feedback loop between the agent and background work: commands run in nowait mode now send a completion signal back to the running agent, and any active wait ends early so the agent can react immediately instead of sleeping through it. Clipboard paste handling is simplified and made consistent across all surfaces. A visual regression in the chat panel is fixed, and sessions with partially invalid stored history are now sanitized on load rather than failing.

### Highlights You Will Notice

- **Background command completion notifications**
  - when the agent runs a command in `nowait` mode, or you skip waiting for a long-running command, GyShell now automatically notifies the agent when that command finishes
  - the agent receives a structured notification with the terminal and `history_command_match_id` so it can inspect the output at the right moment — without polling or guessing
  - if the agent is sitting in an active `wait` call when the notification arrives, the wait ends early and the agent immediately processes the completion instead of sleeping out the full timer
  - the task guard now also warns the agent if there are still-unfinished background commands before it wraps up a turn, so it does not declare victory while work is still running
- **Simpler, more reliable clipboard paste**
  - large clipboard pastes are now inserted directly as plain text in all surfaces: desktop, TUI, and mobile-web
  - the previous system saved large pastes as temporary files on disk and referenced them by path, which added latency, left temp files behind, and created subtle surface-specific inconsistencies
  - pasting now just works the same everywhere
- **ASSISTANT label always appears at the start of a turn**
  - fixed a display regression in both Classic and Seamless modes where the `ASSISTANT` role label was skipped if the turn opened with a tool call or terminal command rather than a text message
  - the label is now anchored to the first item of each assistant turn regardless of type
- **Session stability with malformed history**
  - sessions that contain partially invalid or undeserializable stored messages now load cleanly instead of failing or behaving unpredictably
  - the invalid entries are silently filtered on load, at persistence, and on rollback without corrupting the rest of the session

### What This Means in Practice

- Nowait and background-mode commands are no longer fire-and-forget from the agent's perspective — it gets called back when the work finishes and can continue the task naturally.
- Workflows where you skip waiting on a slow build or long script now close the loop automatically.
- Pasting a large block of code, a log dump, or any long text is simpler and identical across every surface.
- The chat transcript is visually correct again in both display modes — you can always tell at a glance where each assistant turn begins.
- Old sessions that would silently misbehave due to stale or corrupted entries in history now open and run normally.

## 中文

v1.4.2 收紧了 Agent 与后台任务之间的反馈回路：以 nowait 模式运行的命令完成后，现在会自动通知正在运行的 Agent；如果 Agent 正在等待中，等待会提前结束，让 Agent 能立刻响应，而不是睡过去。剪贴板粘贴逻辑在所有端都得到了简化和统一。对话面板的一个视觉回归问题已修复，包含部分无效历史记录的会话现在会在加载时自动清理，而不是直接报错。

### 你会直接感知到的升级

- **后台命令完成通知**
  - Agent 以 `nowait` 模式运行命令，或你跳过等待时，GyShell 现在会在命令完成后自动通知 Agent
  - Agent 会收到一条结构化通知，包含终端信息和 `history_command_match_id`，让它可以在恰当的时机检查输出——不需要轮询，不需要猜测
  - 如果 Agent 此时正在执行 `wait` 等待，通知到达后等待会提前结束，Agent 立刻处理完成事件，而不是睡满整个计时器
  - task guard（任务守卫）在 Agent 结束某一轮之前，也会提醒它还有哪些后台命令尚未完成，避免 Agent 在任务还没跑完时就提前收尾
- **更简单可靠的剪贴板粘贴**
  - 大段剪贴板内容现在在所有端都直接以纯文本插入：桌面端、TUI、Mobile Web 行为完全一致
  - 旧系统会把大段粘贴内容保存为磁盘临时文件并以路径引用，引入了延迟、留下了临时文件，并在各端之间产生细微的行为差异
  - 现在粘贴就是粘贴，到处一样
- **ASSISTANT 标签始终出现在对话轮次起点**
  - 修复了 Classic 和 Seamless 两种模式下的一个显示回归：当一轮 Assistant 回复以工具调用或终端命令开头（而非文字消息）时，`ASSISTANT` 角色标签会被跳过
  - 现在标签始终锚定在每轮 Assistant 回复的第一条消息上，与消息类型无关
- **含损坏历史记录的会话稳定性修复**
  - 包含部分无效或无法反序列化历史记录的会话，现在可以正常加载，不再报错或行为异常
  - 无效条目会在加载、持久化和回滚时静默过滤，不会影响同一会话中的其他记录

### 这次升级带来的实际体验

- 对 Agent 来说，nowait 和后台命令不再是"发出去就不管了"——它会在工作完成时收到回调，可以自然地继续推进任务。
- 等待慢速构建或长脚本时选择跳过，现在会自动形成闭环。
- 粘贴大段代码、日志或任意长文本，在所有端上都更简单，体验完全一致。
- 两种显示模式下的对话记录视觉上重新正确了——一眼就能看出每轮 Assistant 回复从哪里开始。
- 以前因为历史记录里有过期或损坏条目而悄悄出错的旧会话，现在可以正常打开和运行。
