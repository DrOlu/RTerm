# v1.5.3 Release Notes

## English

v1.5.3 is a workspace-control and remote-steering release. The headline additions are **saved workspace layout slots** and **Agent Setting profiles**: you can now keep a few reusable panel layouts and a few reusable agent/runtime configuration snapshots instead of rebuilding them by hand. The mobile-web companion also grows into a more complete steering surface, with richer session status, approval signals, branching/rollback controls, task-completion toasts, and a read-only terminal panel that can create, close, monitor, and reconnect terminal tabs. This release also formally deprecates the desktop-bundled `gyll` CLI/TUI runtime: desktop packages no longer bundle or install it, and startup now cleans up old desktop-managed launchers.

### Highlights You Will Notice

- **Saved workspace layout slots**
  - the workspace rail now supports up to **three saved layout slots**. Save the current layout into the first free numbered slot, click a number to restore it, and right-click a slot to overwrite it with the current layout or delete it
  - saved layouts store the full v2 layout tree plus the legacy `panelOrder` / `panelSizes` projection, so advanced split layouts, panel kinds, focused panel, panel tab bindings, and active tabs can be restored together
  - applying a saved layout clears pending layout-save timers, marks that saved slot as active, restores saved active tabs instead of stale global active tabs, and preserves placeholder panels when a saved tab binding points to a terminal/chat tab that no longer exists
  - changing the current layout clears the active saved-layout marker, so the UI does not pretend you are still exactly on a saved layout after resizing, splitting, dragging, or closing panels
  - saved-layout controls are hidden in non-persistent layout contexts, and malformed / duplicate persisted slots are normalized defensively
- **Agent Setting profiles**
  - Settings now has numbered Agent Setting slots, with save, apply, overwrite, and delete actions. A slot captures the operational profile you are currently using, not just the model name
  - each snapshot includes the active model profile, command policy mode and allow/deny/ask lists, built-in tool enablement, MCP server enablement, skill enablement, memory enabled state, recursion limit, and workflow experimental flags
  - up to **five** Agent Setting profiles can be saved. Slot numbers stay stable, deleted numbers are reused, and concurrent saves are serialized so two saves do not collide into the same slot
  - profile-specific `memory.md` files are supported under the active Agent Setting profile. Saving or overwriting a profile copies the currently active memory into that profile's memory file, and applying a profile switches memory reads/writes to that profile
  - applying a profile is conservative: missing model profiles produce a warning and keep the current model profile; missing tools or skills are not recreated; new tools or skills absent from the saved snapshot keep their current state
  - profile operations broadcast updated settings, command-policy lists, MCP tools, built-in tools, skills, and memory snapshots to connected clients, so desktop, mobile-web, and gateway clients stay in sync
- **Mobile-web becomes a fuller remote steering surface**
  - the mobile UI is reorganized around **Sessions**, **Terminals**, and **Settings**, with clearer navigation between the session list, conversation view, terminal view, and settings sub-pages
  - the session list now shows compact status signals for approvals, errors, thinking, replies, tool calls, file edits, sub-tools, foreground/background commands, compaction, running, and done states
  - pending approvals are surfaced globally with a badge strip that jumps directly to the first blocked session, so a remote run does not silently wait for a command/tool decision
  - mobile conversations now expose approval replies, rollback, branch-from-message, profile switching, locked-profile display, image attachments, and richer detail sheets from the phone browser
  - session cleanup is faster on touch devices with a conservative swipe-to-delete gesture, and task completion now raises in-page toasts plus a short document-title flash
  - the gateway connection flow now stores auto-connect intent, uses heartbeat checks, handles offline/online browser events, and performs bounded auto-reconnect attempts with backoff
- **Read-only mobile terminal control**
  - the mobile Terminals tab polls `terminal:getBufferDelta` and keeps a rolling output tail per terminal, with unread indicators on tabs that received output while you were viewing another terminal
  - from mobile-web you can create a local terminal, create an SSH terminal from desktop-saved SSH connection settings, close terminal tabs while protecting the last tab, and refresh terminal output manually
  - exited SSH terminal tabs can be reconnected from mobile-web through the same backend reconnect RPC used by desktop
  - the terminal surface is intentionally read-only: it is designed for observing and steering via chat rather than turning the phone browser into a full TTY
- **Desktop-bundled `gyll` CLI/TUI is deprecated**
  - desktop packages no longer prepare, bundle, or install the `gyll` / `gyll-tui` CLI runtime, and build scripts no longer generate platform CLI binaries or package a `resources/cli` payload
  - new desktop installs do not add launcher files or modify shell profiles
  - on startup, the desktop app removes old desktop-managed `gyll` launchers only when they contain the legacy `GYLL_BIN` marker. Unrelated user files are preserved, and existing shell profile PATH blocks are intentionally left untouched
  - the historical TUI source remains in the repository as deprecated and unsupported source compatibility, not as a desktop-distributed surface
- **Font/readability polish**
  - the renderer now uses native font smoothing instead of forcing grayscale antialiasing, making small UI text less thin on light themes
  - xterm now sets a medium normal font weight and bold font weight explicitly, improving terminal readability without depending on platform defaults

### Under the Hood

- **Settings schema and normalization**
  - backend settings advance to schema v4 with a new `agentSettings` state. Migration initializes empty Agent Setting profiles and normalizes malformed profile payloads, stale active ids, duplicate slots, bad command-policy lists, and incomplete experimental flags
  - layout settings now persist `savedLayouts` and `activeSavedLayoutId` alongside the existing layout tree and legacy layout projections
- **Agent Setting runtime service**
  - a new `AgentSettingProfileService` owns serialized save/apply/overwrite/delete mutations and builds operation results that include every client-facing snapshot needed after a settings switch
  - command-policy stores now support whole-list replacement, MCP runtime supports batch enablement, and memory runtime APIs accept an optional Agent Setting profile id
  - desktop Electron IPC, Electron gateway IPC, gybackend, WebSocket gateway, and mobile-web were updated to expose the new Agent Setting operations; the TUI now consumes the related settings/profile, command-policy, and memory broadcasts where relevant
- **Mobile-web helpers and terminal polling**
  - mobile-web extracts normalization/status helpers for skills, tools, connections, command-policy lists, memory snapshots, Agent Setting profiles, session status, and SSH config construction
  - `useTerminalBuffer` keeps offset-based output tails per terminal, trims to a bounded character window, tracks seen offsets, and resets state on disconnect or terminal removal
- **Packaged mobile-web runtime refreshed**
  - the Electron-packaged mobile-web assets were regenerated so desktop builds serve the new steering UI and terminal controls without requiring a separate mobile-web development server
- **Deprecated CLI cleanup**
  - `CliInstallService`, CLI runtime preparation, Linux CLI validation, packaged CLI resources, and CLI binary build scripts were removed from the desktop packaging path
  - `DeprecatedCliCleanupService` performs marker-based cleanup for old POSIX and Windows launchers, including WindowsApps launchers and the fallback `~/.gyll/bin` path, while leaving unrelated files and shell profile blocks alone
- **Tests**
  - new extreme specs cover saved layout normalization and LayoutStore behavior, Agent Setting normalization and profile service behavior, AppStore settings/profile updates, mobile session status derivation, mobile terminal buffer polling, terminal-panel decisions, session-browser swipe/status behavior, and deprecated desktop CLI cleanup/build guards

### What This Means in Practice

- You can keep a few workspace shapes ready for different modes of work and switch between them without rebuilding split panels and tab placement every time.
- You can keep a few complete agent operating modes ready, including model profile, policy, tools, skills, memory, recursion, and experimental behavior.
- Your phone browser is now useful for real remote supervision: find blocked approvals, branch or roll back a run, inspect task progress, check terminal output, reconnect SSH, and get completion feedback.
- New desktop installs no longer include `gyll`; users upgrading from older desktop builds get stale desktop-managed launchers cleaned up without the app rewriting shell profiles.

## 中文

v1.5.3 是一次聚焦于工作区控制与远程接管能力的更新。最主要的新增是 **已保存工作区布局槽位** 和 **Agent Setting 配置档**：你现在可以保存几套常用面板布局，也可以保存几套常用 Agent/运行时配置，不必每次手动重新搭。Mobile Web 伴随端也进一步变成更完整的远程操控界面，新增更丰富的会话状态、审批提醒、分支/回滚控制、任务完成 toast，以及可只读查看输出、创建、关闭、监控、重连 terminal tab 的终端面板。本次更新还正式废弃桌面版内置的 `gyll` CLI/TUI 运行时：桌面安装包不再内置或安装它，启动时会清理旧版本由桌面端生成的 launcher。

### 你会直接感知到的升级

- **已保存工作区布局槽位**
  - 工作区 Rail 现在支持最多 **3 个已保存布局槽位**。你可以把当前布局保存到第一个空编号槽，点击编号恢复布局，右键某个槽位可用当前布局覆盖它或删除它
  - 已保存布局会保存完整 v2 布局树，同时保留旧的 `panelOrder` / `panelSizes` 投影，因此复杂分屏、panel 类型、焦点 panel、panel tab 绑定和活跃 tab 都可以一起恢复
  - 应用已保存布局时会清掉待执行的布局保存定时器，把该槽标记为活跃，并优先恢复布局中保存的活跃 tab，而不是使用过期的全局活跃 tab；当保存的 tab 绑定指向已经不存在的 terminal/chat tab 时，也会保留占位 panel
  - 只要你继续修改当前布局，活跃的已保存布局标记就会被清空，避免 UI 在 resize、split、拖拽或关闭 panel 后仍误显示“当前就是某个已保存布局”
  - 在不持久化布局的上下文中不会展示保存布局控件；格式错误、重复或过期的持久化槽位会被防御性规范化
- **Agent Setting 配置档**
  - Settings 中新增编号式 Agent Setting 槽位，支持保存、应用、覆盖和删除。它保存的是当前使用中的完整运行配置，而不仅仅是模型名称
  - 每个快照包含：当前模型 Profile、命令策略模式、allow/deny/ask 规则列表、内置工具开关、MCP server 开关、Skill 开关、memory 开关、递归限制，以及 workflow experimental flags
  - 最多可保存 **5 个** Agent Setting 配置档。槽位编号保持稳定，删除后的编号会被复用，并发保存也会串行执行，避免两个保存操作抢同一个槽位
  - 支持按活跃 Agent Setting profile 隔离 `memory.md`。保存或覆盖配置档时，会把当前活跃 memory 复制到该配置档的 memory 文件；应用配置档后，memory 的读写也会切到该配置档
  - 应用配置档时会保守处理差异：如果保存的模型 Profile 已不存在，会给出警告并保留当前模型 Profile；已不存在的工具或 Skill 不会被重新创建；快照里没有的新工具或新 Skill 会保留当前状态
  - 配置档操作会广播更新后的 settings、命令策略列表、MCP 工具、内置工具、Skills 与 memory 快照，让桌面端、Mobile Web 和网关客户端保持同步
- **Mobile Web 变成更完整的远程操控界面**
  - 移动端 UI 重新围绕 **Sessions**、**Terminals** 和 **Settings** 组织，能更清晰地在会话列表、对话视图、终端视图和设置子页面之间切换
  - 会话列表现在会展示紧凑状态信号，覆盖等待审批、错误、思考中、回复中、工具调用、文件编辑、子工具、前台/后台命令、压缩中、运行中和已完成等状态
  - 待审批任务会通过全局 badge strip 显示，并可直接跳转到第一个被阻塞的会话，远程运行不会悄悄卡在命令/工具决策上
  - 手机浏览器中的对话视图现在支持审批回复、回滚、从消息分支、Profile 切换、锁定 Profile 展示、图片附件，以及更丰富的单轮详情页
  - 触屏清理会话更快：会话列表新增保守的左滑删除手势；任务完成时会显示页面内 toast，并短暂闪烁网页标题
  - 网关连接流程会保存自动连接意图，使用 heartbeat 检测，响应浏览器 offline/online 事件，并在断线时执行带退避和次数上限的自动重连
- **移动端只读终端控制**
  - Mobile Web 的 Terminals tab 现在会轮询 `terminal:getBufferDelta`，为每个 terminal 保留滚动输出尾部；如果你正在看另一个 terminal，收到新输出的 tab 会显示未读提示
  - 你可以在 Mobile Web 中创建本地 terminal、基于桌面端已保存的 SSH 连接创建 SSH terminal、关闭 terminal tab（保护最后一个 tab 不被关闭），并手动刷新终端输出
  - 已退出的 SSH terminal tab 可以从 Mobile Web 通过与桌面端相同的后端 reconnect RPC 原地重连
  - 该终端视图刻意保持只读：它的定位是观察状态并通过对话继续接管，而不是把手机浏览器做成完整 TTY
- **桌面内置 `gyll` CLI/TUI 已废弃**
  - 桌面安装包不再准备、打包或安装 `gyll` / `gyll-tui` CLI runtime，构建脚本也不再生成平台 CLI 二进制或打包 `resources/cli`
  - 新安装桌面版不会添加 launcher 文件，也不会修改 shell profiles
  - 桌面端启动时只会删除包含旧版 `GYLL_BIN` 标记的、由桌面端管理过的 `gyll` launcher；无关用户文件会被保留，已有 shell profile PATH block 也会被有意保留
  - 仓库中的历史 TUI 源码仅作为已废弃、无支持的源码兼容内容保留，不再是桌面安装包分发入口
- **字体与可读性打磨**
  - 渲染层不再强制使用 grayscale antialiasing，而是改回原生字体平滑，让浅色主题下的小字号 UI 文字不再过细
  - xterm 显式设置中等普通字重和 bold 字重，减少不同平台默认字体渲染差异，提升终端可读性

### 底层改动

- **设置 schema 与规范化**
  - 后端设置升级到 schema v4，新增 `agentSettings` 状态。迁移会初始化空的 Agent Setting 配置档，并规范化损坏的 profile 负载、过期 active id、重复槽位、异常命令策略列表和缺失的 experimental flags
  - layout 设置现在会在现有布局树和旧布局投影之外，额外持久化 `savedLayouts` 与 `activeSavedLayoutId`
- **Agent Setting 运行时服务**
  - 新增 `AgentSettingProfileService`，统一负责串行化的 save/apply/overwrite/delete 操作，并生成配置切换后各客户端需要的完整结果快照
  - command-policy store 支持整表替换，MCP runtime 支持批量启停，memory runtime API 支持可选 Agent Setting profile id
  - 桌面 Electron IPC、Electron gateway IPC、gybackend、WebSocket gateway 与 Mobile Web 已接入新的 Agent Setting 操作；TUI 会消费相关 settings/profile、命令策略与 memory 广播
- **Mobile Web 辅助层与终端轮询**
  - Mobile Web 抽取了 skills、tools、connections、命令策略列表、memory 快照、Agent Setting 配置档、会话状态和 SSH config 构造等规范化/状态辅助函数
  - `useTerminalBuffer` 按 terminal 保存基于 offset 的输出尾部，限制字符窗口，跟踪已读 offset，并在断线或 terminal 被移除时重置状态
- **内置 Mobile Web 运行时已刷新**
  - Electron 打包内置的 Mobile Web 产物已重新生成，因此桌面安装包直接托管的移动端页面也会包含新的远程操控 UI 与终端控制能力，不需要单独启动 mobile-web 开发服务
- **已废弃 CLI 清理**
  - `CliInstallService`、CLI runtime 准备流程、Linux CLI 校验、打包 CLI resources，以及 CLI 二进制构建脚本都已从桌面打包路径移除
  - `DeprecatedCliCleanupService` 对旧 POSIX / Windows launcher 做基于标记的清理，包括 WindowsApps launcher 与 fallback 的 `~/.gyll/bin` 路径，同时保留无关文件和 shell profile block
- **测试**
  - 新增的 extreme 规格覆盖了已保存布局规范化和 LayoutStore 行为、Agent Setting 规范化和 profile service 行为、AppStore settings/profile 更新、移动端会话状态推导、移动端 terminal buffer 轮询、terminal panel 决策、session browser 左滑/状态行为，以及桌面 CLI 弃用后的清理和构建守卫

### 这次升级带来的实际体验

- 你可以为不同工作模式保存几套工作区形态，并在它们之间切换，不必每次重新搭分屏和 tab 位置。
- 你可以保存几套完整 Agent 运行模式，包括模型 Profile、策略、工具、Skills、memory、递归和实验行为。
- 手机浏览器现在可以承担真正的远程监督：发现等待审批的任务、分支或回滚运行、查看进度、检查终端输出、重连 SSH，并收到任务完成反馈。
- 新安装桌面版不再包含 `gyll`；从旧桌面版本升级的用户会清理陈旧的桌面端 launcher，但不会被应用重写 shell profiles。
