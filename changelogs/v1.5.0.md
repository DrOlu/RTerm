# v1.5.0 Release Notes

## English

v1.5.0 is a feature-and-reliability release centered on the chat workflow, multilingual input, and layout persistence. The headline addition is **chat branching** — you can now fork a new conversation from any assistant message and explore a different direction without losing the original thread. Alongside it, this release fixes an IME issue where confirming a CJK/Japanese/Korean candidate with Enter would accidentally submit the message, hardens SSH session teardown so exits are reported exactly once, smooths out chat auto-scroll, and — most importantly for Windows users — makes panel-layout state survive a window close so your workspace comes back the way you left it. The Resource Monitor also gains support for **Enrigin (江原) ECU accelerators**, so domestic GPU/accelerator hosts now report telemetry next to NVIDIA cards.

### Highlights You Will Notice

- **Branch a new session from any assistant message ("branch from here")**
  - assistant message groups now show a branch button (the `GitBranch` icon) next to the copy action. Clicking it forks a brand-new session that copies both the agent history and the on-screen UI history *up through the selected message*
  - the new session opens in the same panel and is titled `<source>_branch`, so the fork is easy to spot in the tab and history list while the original conversation stays completely intact
  - if you branch from a UI-only message (one with no backing agent turn), GyShell falls back to the nearest preceding agent anchor so the fork still lands on a coherent point in the conversation
  - branching is only offered when it can actually succeed — the source message must be a real, finished backend turn (not streaming, not mid-think) — and if the source or target can't be resolved the operation rolls back cleanly and leaves your current session untouched
  - internally this reuses the same `prepareMessagesBeforeCutIndex` logic as rollback, so branching and rollback stay consistent and rollback behavior is unchanged
- **IME composition no longer submits your message by accident**
  - when composing with an input method (Chinese, Japanese, Korean, etc.), the Enter that *confirms a candidate* is no longer treated as "send". A new composition tracker watches `compositionstart` / `compositionend` and adds a short post-composition Enter-suppression window
  - this applies both to the chat input (`RichInput`) and to the AI terminal command draft box (`TerminalCommandDraft`), so candidate selection and message/command submission no longer collide
- **Chat auto-scroll respects you reading earlier messages**
  - the view now snaps to the bottom only **once per new user message**, instead of forcing scroll-to-bottom on every render
  - that means you can scroll up to re-read context while the assistant is still replying, and the view won't keep yanking you back down
- **Panel layout is restored reliably after closing the window (Windows fix)**
  - layout changes were previously saved through a debounced async IPC that could be dropped during window teardown — most reliably on Windows — so panel sizes and ordering sometimes weren't restored on the next launch
  - GyShell now has a blocking, synchronous settings-flush path that persists the pending layout before the window unloads, with an async fallback, so your panel sizes/order come back as you left them
  - a startup race in the split-pane view that could persist the *initial* sizes (mistaking the panel group's first layout callbacks for a real user resize) is fixed, so a freshly opened window no longer overwrites your saved sizes
- **More predictable split-pane resizing**
  - dragging a split divider now clamps each pane to its child minimum size, so you can't accidentally crush a panel below its usable width/height
- **Tidier chat-history titles**
  - history titles now wrap to a clean two-line CSS clamp instead of being hard-truncated in JavaScript, so longer titles read better without being abruptly cut
- **Resource Monitor now reads Enrigin (江原) ECU accelerators**
  - on Linux hosts the monitor probes `ersmi` (the Enrigin system management tool) in addition to `nvidia-smi`, so Enrigin ECU cards show up in the GPU section right beside NVIDIA GPUs
  - it collects both `ersmi -q` (device inventory) and `ersmi --dmon` (live utilization) and merges them per device, with a non-interactive `sudo -n` fallback for hosts where `ersmi` needs elevated access
  - each accelerator now surfaces **power draw vs. power cap** (with a dedicated power meter), power/DPM state, and memory clock, in addition to the existing utilization / memory / temperature readouts
  - the panel gracefully degrades when a metric is missing — dmon-only devices still get a stable `Enrigin ECU N` name and derive used memory from the reported usage percentage

### Under the Hood

- **SSH exits are reported exactly once**
  - SSH session teardown is consolidated into a single `emitExitOnce` path guarded by an `exitEmitted` flag, and the backend now also handles client `end` / `close` events
  - on errors and connect failures, port forwards, SFTP handles, and chunked-transfer sessions are released and exit callbacks fire exactly once — no missed cleanup and no duplicate exit notifications
- **Layout persistence is guarded and refactored**
  - the new synchronous channel is restricted to layout-only payloads by `normalizeSyncSettingsPatch`, which rejects any other settings keys (gateway, recursion limit, etc.) so the blocking IPC can't be used to write unrelated settings
  - `saveLayout` is refactored into shared helpers (`buildLayoutSettingsPayload`, `applyLayoutSettingsPayload`, `shouldPersistLayout`) used by both the async and synchronous flush paths
- **GPU telemetry parsing is vendor-agnostic**
  - the Linux GPU collector is refactored into a `parseLinuxGpuOutput` dispatcher that fans out to the NVIDIA and ERSMI parsers, and the section command and `GpuSnapshot` type are extended with the new power/clock fields so both backends share one shape
- **Tests**
  - new extreme specs cover chat branching, IME composition, chat-history presentation, the sync settings-patch normalizer, synchronous layout flush, and Enrigin ERSMI parsing (query + dmon merge, dmon-only fallback, sudo fallback in the section command); chat virtual-list coverage is extended, and the new specs are wired into the extreme test scripts

### What This Means in Practice

- You can explore "what if" directions in a conversation safely: branch from a good answer, try a different follow-up, and the original thread is still there to return to.
- Typing in Chinese/Japanese/Korean (and other IME-based input) in chat and in the terminal command draft is no longer interrupted by Enter prematurely sending your message.
- You can read back through a long reply without the chat constantly snapping to the bottom.
- On Windows especially, closing and reopening GyShell now restores your panel layout faithfully instead of resetting sizes or order.
- If you run on Enrigin (江原) ECU hardware, the Resource Monitor now shows those accelerators — including power draw, power cap, and clocks — right alongside NVIDIA GPUs.

## 中文

v1.5.0 是一次聚焦于聊天工作流、多语言输入与布局持久化的功能与可靠性更新。最大的新增是 **聊天分支（chat branching）**——你现在可以从任意一条助手消息分叉出一个新会话，去探索不同的方向，而完全不影响原始对话线。除此之外，本次更新还修复了一个输入法相关的问题：以往用 Enter 确认中文/日文/韩文候选词时会误触发消息发送；同时收紧了 SSH 会话的拆除逻辑，使退出事件只上报一次；优化了聊天自动滚动；并且——对 Windows 用户尤其重要——让面板布局状态能扛过窗口关闭，下次打开时工作区会保持你离开时的样子。资源监控还新增了对 **江原（Enrigin）ECU 加速卡** 的支持，让国产 GPU/加速卡主机能与 NVIDIA 显卡并排上报遥测数据。

### 你会直接感知到的升级

- **从任意助手消息分叉出新会话（"branch from here"）**
  - 助手消息组现在会在复制按钮旁边显示一个分支按钮（`GitBranch` 图标）。点击它会分叉出一个全新会话，把 agent 历史和屏幕上的 UI 历史 **复制到所选消息为止**
  - 新会话会在同一面板中打开，并以 `<source>_branch` 命名，因此在标签页和历史列表里都很容易识别这个分叉，而原始对话保持完全不变
  - 如果你从一条仅存在于 UI、没有对应 agent 轮次的消息上分支，GyShell 会回退到最近的前一个 agent 锚点，让分叉仍落在对话中一个连贯的位置
  - 只有在真正能成功时才会提供分支——源消息必须是一条真实且已完成的后端轮次（不在流式输出中、也不在思考中）——并且当源或目标无法解析时，操作会干净地回滚，不会动到你当前的会话
  - 在内部，这复用了与回滚相同的 `prepareMessagesBeforeCutIndex` 逻辑，因此分支与回滚保持一致，回滚行为也未发生变化
- **输入法组词不再误发消息**
  - 在使用输入法（中文、日文、韩文等）组词时，用来 **确认候选词** 的 Enter 不再被当作"发送"。新增的组词追踪器会监听 `compositionstart` / `compositionend`，并加入一个短暂的"组词结束后 Enter 抑制"窗口
  - 该处理同时作用于聊天输入框（`RichInput`）和 AI 终端命令草稿框（`TerminalCommandDraft`），从此候选词选择与消息/命令提交不再相互冲突
- **聊天自动滚动会尊重你正在回看更早的消息**
  - 视图现在 **每条新用户消息只吸底一次**，而不是在每次渲染时都强制滚动到底部
  - 这意味着你可以在助手仍在回复时向上滚动回看上下文，视图不会再不停地把你拽回底部
- **关闭窗口后能可靠恢复面板布局（Windows 修复）**
  - 以往布局变更是通过一个去抖（debounced）的异步 IPC 保存的，而它可能在窗口拆除时被丢弃——在 Windows 上尤为明显——因此面板尺寸和顺序有时在下次启动时无法恢复
  - GyShell 现在拥有一条阻塞式的同步设置刷写路径，会在窗口卸载前把待保存的布局持久化，并带有异步回退，因此你的面板尺寸/顺序会照你离开时的样子恢复
  - 修复了分屏视图中一个可能持久化 **初始** 尺寸的启动竞态（它会把面板组首次的布局回调误认为是用户真正的 resize），因此新打开的窗口不会再覆盖你保存的尺寸
- **更可预期的分屏拖拽**
  - 拖动分隔条时现在会把每个分屏钳制到其子项最小尺寸，因此你不会再不小心把某个面板压到不可用的宽/高以下
- **更整洁的聊天历史标题**
  - 历史标题现在用干净的两行 CSS 截断换行，而不再用 JavaScript 硬截断，因此较长的标题更易读，也不会被生硬地切断
- **资源监控现已读取江原（Enrigin）ECU 加速卡**
  - 在 Linux 主机上，监控除了 `nvidia-smi` 之外还会探测 `ersmi`（江原的系统管理工具），因此江原 ECU 卡会与 NVIDIA GPU 一起出现在 GPU 区域
  - 它同时采集 `ersmi -q`（设备清单）和 `ersmi --dmon`（实时利用率）并按设备合并；对于需要提权访问的主机，提供了非交互式 `sudo -n` 回退
  - 除了原有的利用率 / 显存 / 温度读数外，每张加速卡现在还会展示 **功耗与功耗上限**（带专门的功耗条）、电源/DPM 状态以及显存频率
  - 当某项指标缺失时面板会优雅降级——仅有 dmon 数据的设备仍会获得稳定的 `Enrigin ECU N` 名称，并根据上报的占用百分比推算已用显存

### 底层改动

- **SSH 退出只上报一次**
  - SSH 会话拆除被收敛到单一的 `emitExitOnce` 路径，由 `exitEmitted` 标志守护，后端现在还会处理客户端的 `end` / `close` 事件
  - 在出错和连接失败时，端口转发、SFTP 句柄以及分块传输会话都会被释放，退出回调恰好触发一次——既不会漏掉清理，也不会重复通知退出
- **布局持久化的守护与重构**
  - 新的同步通道由 `normalizeSyncSettingsPatch` 限制为仅接受布局负载，它会拒绝其它任何设置键（gateway、递归上限等），因此这条阻塞式 IPC 无法被用来写入无关设置
  - `saveLayout` 被重构为共享辅助函数（`buildLayoutSettingsPayload`、`applyLayoutSettingsPayload`、`shouldPersistLayout`），供异步与同步刷写两条路径共用
- **GPU 遥测解析与厂商无关**
  - Linux 的 GPU 采集被重构为 `parseLinuxGpuOutput` 分发器，分别下发给 NVIDIA 与 ERSMI 两个解析器；采集命令与 `GpuSnapshot` 类型也扩展了新的功耗/频率字段，使两套后端共用同一数据形状
- **测试**
  - 新增的 extreme 规格覆盖了聊天分支、输入法组词、聊天历史展示、同步设置补丁规范化、同步布局刷写，以及江原 ERSMI 解析（query + dmon 合并、仅 dmon 时的回退、采集命令中的 sudo 回退）；聊天虚拟列表的测试覆盖也得到了扩展，新规格已接入 extreme 测试脚本

### 这次升级带来的实际体验

- 你可以安全地在一段对话里探索"如果换个方向会怎样"：从一个满意的回答处分支，尝试不同的追问，而原始对话线依然在那里，随时可以回到。
- 在聊天和终端命令草稿框里用中文/日文/韩文（以及其它基于输入法的输入）打字，不再会被 Enter 提前发送消息所打断。
- 你可以从容地回看一段很长的回复，聊天视图不会再不停地吸底。
- 尤其在 Windows 上，关闭并重新打开 GyShell 现在会忠实地恢复你的面板布局，而不是重置尺寸或顺序。
- 如果你运行在江原（Enrigin）ECU 硬件上，资源监控现在会把这些加速卡——包括功耗、功耗上限与频率——与 NVIDIA GPU 并排展示。
