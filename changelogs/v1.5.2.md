# v1.5.2 Release Notes

## English

v1.5.2 is a runtime-reliability release centered on long-running sessions and terminal resilience. The headline addition is an **experimental agent file-transfer tool** — the agent can now copy files asynchronously *between two different machines' terminal tabs* and poll the transfer's progress, with the same task surfacing in the Transfer Tasks panel as user-driven copies. Alongside it, GyShell can now **keep your computer awake while a chat session is running**, **auto-restart a local terminal** when its shell exits so a local tab never goes dead, and offers a **Reconnect action for disconnected SSH terminals**. This release also hardens the agent stream loop against providers that report a tool-call finish without an actual tool-call payload, and polishes the chat/terminal tab indicators and selector menus.

### Highlights You Will Notice

- **Experimental agent file transfer between terminal tabs**
  - the agent gains two new **experimental, off-by-default** tools — `copy_between_tabs` and `read_file_transfer_status` — exposed in Settings under a new **Experimental Tools** group, separate from the standard built-in tools
  - `copy_between_tabs` starts an **asynchronous, copy-only** transfer from one terminal tab to another that is **on a different machine** (cross-machine is enforced; it never cuts, moves, or deletes source files). It returns immediately after queuing, so the agent uses `read_file_transfer_status` (by `transferId`, or list active transfers) to follow progress and confirm the final result
  - the default conflict strategy is **rename** (keep both); `overwrite` is only used when you explicitly ask for replacement
  - when a transfer reaches a terminal state, the agent receives a `file_transfer_finished` notification; on `error`/`cancelled` it is warned that the target directory may contain **incomplete files** and to verify/retry/clean up before treating them as complete
  - agent transfers share the same engine and UI as manual transfers: the **Transfer Tasks** panel now tags each task with an **AGENT / USER** origin badge and shows a new **Scanning** phase, and transfers cancel cleanly when a participating tab is closed
- **Prevent sleep while a session is running**
  - a new **Prevent Sleep While Running** toggle (Settings) keeps the computer awake whenever any chat session is actively running, so long autonomous tasks don't stall when you step away
  - the screen may still turn off — only idle *system* sleep is blocked — and the block is released automatically as soon as all runs finish or the setting is disabled
- **Local terminals stay alive when their shell exits**
  - if a **local** terminal's shell process exits (you type `exit`, or it crashes), GyShell now **respawns a fresh shell in the same tab** instead of leaving a dead "exited" tab behind, preserving the tab, its title, and its size
  - newly opened local terminals are also ready to type into immediately on macOS/Linux (the brief "initializing" state is now Windows-only, where it is actually needed)
- **Reconnect disconnected SSH terminals**
  - a disconnected (exited) **SSH** tab now offers a **Reconnect** action in its right-click menu that re-establishes the connection using the tab's saved SSH config, reusing the same tab instead of forcing you to open a new one
- **Clearer chat & terminal tab indicators**
  - chat tabs now show a small green **"running" dot** while that session is busy, matching the runtime-state dot terminal tabs already use, so you can see at a glance which conversation is still working
  - tab styling is unified through shared CSS variables (font weight, spacing, status-dot size/color) and the close (×) buttons were refined for a consistent, subtler look across chat, terminal, and the compact tab selector
- **More readable selector & context menus**
  - dropdown and right-click menu items are no longer forced to UPPERCASE with wide letter-spacing — they now render in normal case with tighter padding and better line-height, so longer labels stay legible

### Under the Hood

- **Malformed tool-call stream recovery**
  - some OpenAI-compatible providers (e.g. GLM-style endpoints) end a stream with `finish_reason: tool_calls` but emit **no actual tool-call payload** — only reasoning metadata, empty generic chunks, or a raw response. GyShell now detects this "empty malformed tool-call finish" across all of those shapes
  - on detection it performs **one non-streaming `invoke` fallback** for the same request before warning or stopping the turn, recovering the response when possible; empty generic chunks are also skipped/merged safely, and usage/model-name extraction now scans raw chunks as a fallback
  - the new `streamed_model_response` helper is covered by extensive extreme specs for the GLM-compatible malformed shapes
- **Shared file-transfer service**
  - a new `FileTransferService` backs both agent and user transfers (scanning, progress, cancellation, conflict strategy), wired through the gateway, WebSocket adapter, Electron IPC, and preload; the file system panel was refactored onto this shared engine, removing its bespoke transfer plumbing
- **Run-state plumbing for sleep blocking**
  - `GatewayService` now publishes an active-run snapshot, and a small `SleepBlockerService` (backed by Electron's `powerSaveBlocker`) starts/stops the system-sleep block based on that state and the user preference
- **Tests**
  - new extreme specs cover the file-transfer service (scanning cancellation, stale snapshots, closed-tab handling), the WebSocket transfer bindings, app-store transfer state, SSH reconnect / local-terminal restart persistence, gateway run-state, the sleep blocker, and the malformed tool-call stream recovery

### What This Means in Practice

- Long autonomous runs no longer get cut short by your machine going to sleep while you're away.
- A local terminal tab keeps working even after its shell exits, and a dropped SSH session can be brought back from the tab's own menu.
- The agent can move files across machines on its own (when you opt into the experimental tools) and reliably check whether the copy actually finished.
- Flaky "tool_calls but nothing sent" responses from certain providers no longer dead-end a turn — GyShell retries once and keeps going.
- You can tell at a glance which chat is still running, and menus/tabs read more cleanly.

## 中文

v1.5.2 是一次聚焦于长时运行会话与终端韧性的运行时可靠性更新。最大的新增是 **试验性的 Agent 文件传输工具**——Agent 现在可以在 *两台不同机器的终端 tab 之间* 异步复制文件，并轮询传输进度，而且这些任务会和用户手动复制一样出现在「传输任务」面板里。除此之外，GyShell 现在可以在 **对话会话运行期间保持电脑唤醒**，在本地终端的 shell 退出时 **自动重启该本地终端**（让本地 tab 永不"假死"），并为 **已断开的 SSH 终端提供"重新连接"操作**。本次更新还加固了 Agent 流式循环，以应对那些"声称以工具调用结束、却没有真正下发工具调用内容"的模型提供方，并打磨了聊天/终端的标签指示器与选择器菜单。

### 你会直接感知到的升级

- **试验性的 Agent 跨 tab 文件传输**
  - Agent 新增两个 **试验性、默认关闭** 的工具——`copy_between_tabs` 与 `read_file_transfer_status`——它们在设置中以全新的 **试验性工具** 分组展示，与标准内置工具分开
  - `copy_between_tabs` 会启动一个 **异步、仅复制** 的传输，把文件从一个终端 tab 复制到 **另一台机器** 上的终端 tab（强制要求跨机器；它绝不剪切、移动或删除源文件）。它在任务入队后立即返回，因此 Agent 需要用 `read_file_transfer_status`（按 `transferId` 查询，或列出活跃传输）来跟踪进度并确认最终结果
  - 默认冲突策略是 **重命名（保留两者）**；只有在你明确要求替换时才会使用 `overwrite`
  - 当传输到达终止状态时，Agent 会收到 `file_transfer_finished` 通知；若状态为 `error`/`cancelled`，会提示目标目录中可能存在 **不完整的文件**，应先核验/重试/清理，再当作完整文件使用
  - Agent 传输与手动传输共用同一套引擎和界面：「传输任务」面板现在会为每个任务标注 **AGENT / 用户** 来源徽标，并新增了 **扫描中** 阶段；当参与传输的 tab 被关闭时，传输会被干净地取消
- **运行期间阻止系统睡眠**
  - 设置中新增 **运行时阻止系统睡眠** 开关：只要有任意对话会话正在运行，就保持电脑唤醒，这样你离开时长时间的自主任务也不会卡住
  - 屏幕仍可以熄灭——被阻止的只是系统 *空闲睡眠*——并且一旦所有运行结束或关闭该设置，阻止会自动解除
- **本地终端在 shell 退出时保持存活**
  - 如果 **本地** 终端的 shell 进程退出（你输入 `exit`，或它崩溃），GyShell 现在会 **在同一个 tab 内重新拉起一个全新的 shell**，而不是留下一个"已退出"的死 tab，并保留该 tab 的标题与尺寸
  - 新打开的本地终端在 macOS/Linux 上现在也能立即输入（短暂的"初始化中"状态现在仅在确实需要它的 Windows 上出现）
- **重新连接已断开的 SSH 终端**
  - 已断开（已退出）的 **SSH** tab 现在会在右键菜单中提供 **重新连接** 操作，它会用该 tab 保存的 SSH 配置重新建立连接，复用同一个 tab，而不必再新开一个
- **更清晰的聊天与终端标签指示器**
  - 聊天 tab 现在会在该会话繁忙时显示一个小小的绿色 **"运行中"圆点**，与终端 tab 一直在用的运行状态圆点保持一致，让你一眼看出哪个对话还在工作
  - 标签样式通过共享 CSS 变量做了统一（字重、间距、状态点尺寸/颜色），关闭（×）按钮也做了打磨，让聊天、终端与紧凑标签选择器三处的外观一致且更克制
- **更易读的选择器与右键菜单**
  - 下拉菜单与右键菜单项不再被强制为大写加宽字距——现在以正常大小写、更紧凑的内边距和更合适的行高渲染，让较长的标签也保持清晰

### 底层改动

- **畸形工具调用流的恢复**
  - 某些 OpenAI 兼容的提供方（例如 GLM 风格的接口）会以 `finish_reason: tool_calls` 结束流，却 **不下发任何真正的工具调用内容**——只有推理元数据、空的 generic chunk，或一个 raw response。GyShell 现在能在所有这些形态下识别出这种"空的畸形工具调用结束"
  - 一旦识别，它会在告警或终止本轮之前，对同一请求执行 **一次非流式 `invoke` 回退**，尽可能恢复出可用响应；空的 generic chunk 也会被安全地跳过/合并，用量与模型名的提取现在还会扫描 raw chunk 作为回退
  - 新增的 `streamed_model_response` 辅助模块针对 GLM 兼容的畸形形态有大量 extreme 规格覆盖
- **共享的文件传输服务**
  - 新增的 `FileTransferService` 同时支撑 Agent 与用户的传输（扫描、进度、取消、冲突策略），并通过网关、WebSocket 适配器、Electron IPC 与 preload 打通；文件系统面板被重构到这套共享引擎上，移除了它自有的传输管线
- **用于睡眠阻止的运行状态管线**
  - `GatewayService` 现在会发布活跃运行的快照，一个小型的 `SleepBlockerService`（基于 Electron 的 `powerSaveBlocker`）会根据该状态与用户偏好启动/停止系统睡眠阻止
- **测试**
  - 新增的 extreme 规格覆盖了文件传输服务（扫描取消、过期快照、关闭 tab 的处理）、WebSocket 传输绑定、应用 store 的传输状态、SSH 重连 / 本地终端重启的持久化、网关运行状态、睡眠阻止器，以及畸形工具调用流的恢复

### 这次升级带来的实际体验

- 长时间的自主运行不会再因为你离开时电脑进入睡眠而被中断。
- 本地终端 tab 即便在 shell 退出后也能继续使用，断开的 SSH 会话也能从 tab 自己的菜单里恢复回来。
- Agent 可以自己在多台机器之间搬运文件（在你启用试验性工具后），并可靠地确认复制是否真的完成。
- 来自某些提供方的"声称 tool_calls、却什么都没发"的不稳定响应，不会再让一轮对话走入死胡同——GyShell 会重试一次并继续。
- 你可以一眼看出哪个对话还在运行，菜单与标签也更清爽易读。
