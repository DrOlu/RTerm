# v2.5.0 Release Notes

## English

v2.5.0 adds the **RTerm plugin system**: anyone can develop a custom plugin and have it **auto-integrate** with RTerm — drop a folder into `plugins/` with a `plugin.json` manifest and a small code file, and RTerm discovers it, loads it, calls its `register(ctx)` with RTerm's services, and the plugin's capabilities (agent tools, event-driven triggers, dashboard panels) appear automatically. No RTerm code changes needed.

### The plugin system (`plugin/pluginRegistry.ts`)

**Manifest (`plugin.json`)** — declares the plugin's name, version, description, entry module, and what it provides (tools, triggers, panels, permissions).

**Registry** — discovers plugins in a scan root (mirroring the FileSkillStore pattern), loads each plugin's entry module (dynamic import), and manages the lifecycle (enable/disable/uninstall, dedupe-by-name, error recording).

**`register(ctx)` lifecycle** — RTerm calls the plugin's `register(ctx)` with a **PluginContext** that lets the plugin:
- **`registerTool`** — add an agent tool the agent can call (name, description, handler).
- **`registerTrigger`** — add an event-driven trigger (pattern/threshold/webhook/schedule).
- **`registerPanel`** — add a dashboard panel.
- **`exec`** — run a command on a host (the agent's policy-gated exec path).
- **`readLedger`** — read RTerm's ledgers (metrics, incidents, etc.).
- **`log`** — write to the RTerm log.

Failed plugins record an error and are excluded from capability lists but kept in the registry for diagnosis. Capability lists (`allTools`, `allTriggers`, `allPanels`) only include enabled, error-free plugins.

### Sample plugin (`plugins/sample-k8s-slo`)

A working example: a Kubernetes SLO tracker that registers two agent tools (`k8s_slo_evaluate`, `k8s_pod_restarts`), a CrashLoopBackOff trigger, and a dashboard panel — demonstrating the full drop-in auto-integration. Verified end-to-end: discovered, loaded, registered, and its tool executes and returns structured SLO data.

### Write your own plugin

```
plugins/
  my-plugin/
    plugin.json    # { name, version, entry, tools, triggers, panels, permissions }
    index.ts       # export function register(ctx) { ... }
```

```ts
export function register(ctx) {
  ctx.registerTool({
    name: 'my_tool',
    description: 'what it does',
    handler: async (args) => ({ result: 'data' }),
  })
  ctx.registerTrigger({ name: 'my-trigger', kind: 'pattern', match: 'ERROR', action: 'alert' })
  ctx.registerPanel('my-panel', async () => '<h3>My Panel</h3>')
}
```

Drop it in `plugins/` and RTerm auto-integrates it — the agent can call `my_tool`, the trigger fires, and the panel renders in the dashboard.
