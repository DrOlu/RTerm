# v1.5.4 Release Notes

## English

v1.5.4 is a reliability and polish release for terminal state handling, SSH stability, agent file tools, and desktop UI edge cases. The main runtime change is that GyShell now treats an existing-but-disconnected terminal tab as a distinct state instead of letting tools operate on stale retained output. Agent tools now receive explicit `terminal_status` details, can reconnect eligible SSH tabs with `reconnect_terminal_tab`, and avoid running commands, sending input, editing files, waiting for idle, or starting transfers through a terminal that is not ready. This release also enables SSH protocol keepalive, splits file mutation into clearer `write_file` and `edit_file` tools, makes the transfer panel easier to read, turns saved layouts into dynamically updated slots, and fixes floating menu placement around compact tabs and `@` mention suggestions.

### Highlights You Will Notice

- **Terminal-aware tool safety and SSH reconnect**
  - terminal tool results now include a structured `terminal_status` block with runtime state, reconnectability, and last exit code when relevant, so retained output from a disconnected tab is not mistaken for a live shell
  - `exec_command`, `write_stdin`, `read_file`, `wait_terminal_idle`, and cross-tab file transfer now check whether the target terminal can run commands, accept input, or expose filesystem operations before doing work
  - `read_terminal_tab` and `read_command_output` still show retained history for disconnected tabs, but they clearly warn that the backend session is no longer connected and that the output may be stale
  - a new model-visible `reconnect_terminal_tab` tool can reconnect an existing disconnected SSH tab that still has its saved SSH config, preserving the same tab id and output buffer
  - reconnect and command execution form model-visible boundaries, so the agent does not batch a reconnect together with later commands or edits before it has verified the new terminal state
- **SSH keepalive for remote terminals**
  - SSH connections now use encrypted protocol keepalive probes with a bounded failure threshold, reducing silent disconnects on idle or NAT-sensitive remote sessions
  - the keepalive behavior is shared by direct SSH connections and jump-host/tunneled connections while preserving the existing connect ready timeout
- **Clearer file mutation tools**
  - the model-facing file mutation surface is now split into `write_file` for full-file create/overwrite and `edit_file` for exact string replacement
  - the user-facing Settings capability remains `create_or_edit`, so existing tool enablement still controls both concrete tools instead of creating two new settings rows
  - the legacy `create_or_edit` route is still supported for compatibility, including older payloads that used an empty `oldString` as a full-file write
  - `edit_file` now rejects empty `oldString` instead of accidentally creating or replacing a whole file; use `write_file` when a full rewrite is intended
- **More stable transfer task panel**
  - the file system transfer panel now groups visible tasks into **This Tab**, **Background**, and **Recent**, with compact summary counts for running, scanning, queued, background, and recently finished tasks
  - agent-origin transfers can remain visible as background work even when they do not involve the active terminal tab, while completed tasks are retained briefly and predictably
  - queued tasks now show a "waiting for transfer slot" message, scanning has its own summary state, routes include source and target terminal labels, and cancel controls keep rows aligned
- **Saved layouts update dynamically**
  - after applying or saving a layout slot, continued layout edits now update that active saved slot automatically instead of clearing the active-slot marker
  - switching from one saved layout to another first saves the currently active slot's latest shape, so edits are not lost when you try another saved workspace
  - the explicit "overwrite with current layout" menu action was removed because active saved layouts now track ongoing changes directly
- **Floating menus behave better near viewport edges**
  - compact panel tab menus are now portaled to `document.body`, flip above the trigger when needed, clamp to the viewport, and keep their width/spacing tokens even outside the panel header
  - drag/reorder events from portaled compact tab menus are bridged back into the workspace, so tab dragging still works from the compact selector
  - `@` mention suggestions in the chat composer now anchor to the mention chip or caret, reposition on scroll/resize, fit inside the viewport, and automatically keep the active keyboard-selected suggestion visible
- **Theme and localization polish**
  - terminal command draft status chips now use theme tokens instead of hard-coded dark backgrounds, improving light-theme contrast
  - queue cards handle long command text and mention chips without flex overflow
  - Chinese Settings wording now uses a localized label for workflow/mechanism text, and new transfer-panel strings are localized in both English and Chinese

### Under the Hood

- **Terminal runtime snapshots and guards**
  - `TerminalService` now exposes `TerminalRuntimeSnapshot` with `runtimeState`, `reconnectable`, `canRunCommand`, `canWrite`, and `canUseFilesystem`
  - shared `terminal_runtime_guard` helpers normalize terminal resolution errors, unavailable-terminal messages, and status headers across terminal, file, wait, read, and transfer tools
  - mentioned terminal context now uses display terminals and injects terminal status alongside recent output, so agent context matches what tools will later enforce
- **Reconnect plumbing**
  - `reconnect_terminal_tab` calls the existing terminal reconnect path and waits up to 45 seconds for the tab to become ready, reporting success, retry failure, continued initialization, or tab closure explicitly
  - disconnected `write_stdin` calls are rejected before command policy evaluation or the action model, so unavailable terminals do not trigger approval/action-model work
- **File tool capability mapping**
  - `write_file`, `edit_file`, and legacy `create_or_edit` all map back to the single `create_or_edit` built-in capability for Settings, enablement summaries, and disabled-tool enforcement
  - prompt text was updated so memory edits, skill edits, and temporary code-file creation point the agent at the correct concrete tool
- **Transfer panel model extraction**
  - transfer task visibility, terminal-status detection, stable sorting, summary counts, and current/background/recent grouping are now isolated in `fileTransferPresentation.ts`
  - new presentation tests cover active-tab transfers, background agent transfers, recent terminal statuses, and stable ordering
- **Layout persistence behavior**
  - `LayoutStore` now updates the active saved-layout snapshot before debounced saves, sync flushes, direct saves, and saved-layout application
  - tests cover dynamic slot mutation, switching between saved layouts, active-tab changes inside saved layouts, and removal of the manual overwrite flow
- **Tests**
  - new and expanded extreme specs cover terminal runtime guards, reconnect behavior, disconnected-tool prechecks, split file mutation tools, file-transfer panel presentation, SSH keepalive connect config, floating menu placement, compact menu portal drag bridging, and active mention suggestion scrolling

### What This Means in Practice

- If an SSH tab disconnects, GyShell is much less likely to act on stale terminal history as though it were live state.
- Remote SSH terminals should survive idle network conditions more reliably.
- The agent has clearer file-editing primitives: full writes and exact replacements are now separated.
- Transfer progress is easier to understand when multiple user and agent transfers are running at once.
- Saved layout slots now behave more like living workspace presets: edit the active preset and it updates with you.
- Compact tab selectors and `@` mention menus are less likely to open off-screen or hide the selected item.

## 中文

v1.5.4 是一次聚焦于终端状态处理、SSH 稳定性、Agent 文件工具和桌面 UI 边界情况的可靠性与打磨更新。最主要的运行时变化是：GyShell 现在会把“tab 还存在，但后端终端已经断开”识别成独立状态，而不是继续让工具对陈旧的保留输出执行操作。Agent 工具现在会收到明确的 `terminal_status`，可以用 `reconnect_terminal_tab` 重连符合条件的 SSH tab，并且会避免在未就绪的 terminal 上运行命令、发送输入、编辑文件、等待 idle 或启动文件传输。本次更新还启用了 SSH 协议 keepalive，把文件修改工具拆分成更清晰的 `write_file` 与 `edit_file`，改进了传输任务面板，让已保存布局槽位支持动态更新，并修复了紧凑 tab 选择器和 `@` 提及建议菜单的浮层定位问题。

### 你会直接感知到的升级

- **终端状态感知的工具保护与 SSH 重连**
  - 终端工具结果现在会包含结构化 `terminal_status`，展示运行状态、是否可重连，以及必要时的最后退出码，避免把断开 tab 的保留输出误判成还活着的 shell
  - `exec_command`、`write_stdin`、`read_file`、`wait_terminal_idle` 和跨 tab 文件传输都会先检查目标 terminal 是否能运行命令、接受输入或执行文件系统操作
  - `read_terminal_tab` 与 `read_command_output` 仍会展示断开 tab 中保留的历史输出，但会明确提示后端会话已断开，输出可能已经陈旧
  - 新增模型可见工具 `reconnect_terminal_tab`，可以重连仍保有 SSH 配置的已断开 SSH tab，并保留原 tab id 与输出缓冲
  - 重连与命令执行现在都是模型可见的边界操作，Agent 不会在同一批工具调用里把重连和后续命令/编辑混在一起执行，而是必须先验证新的终端状态
- **远程终端启用 SSH keepalive**
  - SSH 连接现在会发送加密协议 keepalive，并设置有界失败阈值，降低空闲连接或 NAT 敏感网络中静默断线的概率
  - 直连 SSH 与 jump host / tunnel 连接共用同一 keepalive 行为，同时保留原有连接 ready timeout
- **更清晰的文件修改工具**
  - 面向模型的文件修改工具拆成 `write_file` 与 `edit_file`：前者用于创建或整文件覆盖，后者用于精确字符串替换
  - 面向用户的 Settings 能力仍然叫 `create_or_edit`，因此现有工具开关仍会同时控制这两个具体工具，不会多出两行设置
  - 旧的 `create_or_edit` 路径继续保留兼容，包括过去用空 `oldString` 表示整文件写入的旧 payload
  - `edit_file` 现在会拒绝空 `oldString`，避免意外创建或覆盖整个文件；如果目标是整文件重写，应使用 `write_file`
- **更稳定易读的传输任务面板**
  - 文件系统传输任务面板现在会按 **当前 Tab**、**后台**、**最近完成** 分组，并在标题上显示进行中、扫描中、排队中、后台和最近完成的紧凑计数
  - Agent 发起的传输即使不涉及当前 terminal tab，也能作为后台任务继续显示；已完成任务会短暂、可预测地保留
  - 排队任务现在会展示“等待传输通道”，扫描也有独立摘要状态；传输路线会包含源/目标 terminal 标签，取消按钮也会保持行布局稳定
- **已保存布局会动态更新**
  - 应用或保存某个布局槽位后，如果继续修改布局，当前活跃槽位会自动更新，而不是清空活跃槽标记
  - 从一个已保存布局切换到另一个之前，会先保存当前活跃槽位的最新形态，避免尝试其它工作区时丢掉刚刚做的布局调整
  - 由于活跃布局现在会随编辑自动跟踪，右键菜单中的“用当前布局覆盖”操作已移除
- **浮层菜单在视口边缘更可靠**
  - 紧凑 panel tab 菜单现在会 portal 到 `document.body`，必要时自动向上展开，钳制在视口内，并在离开 panel header 后仍保留宽度与间距变量
  - 从 portaled 紧凑 tab 菜单发起的拖拽/排序事件会桥接回工作区，因此仍可从紧凑选择器里拖动 tab
  - 聊天输入框的 `@` 提及建议现在会锚定到提及标签或光标位置，滚动/resize 时重新定位，保持在视口内，并自动让键盘选中的建议项保持可见
- **主题与本地化打磨**
  - 终端命令草稿状态标签改用主题 token，不再使用写死的深色背景，浅色主题下对比度更自然
  - 队列卡片现在能处理较长命令文本和 mention 标签，不再因为 flex 布局而溢出
  - 中文 Settings 中的 Workflow 文案改成本地化表达，新增的传输任务文案也已补齐中英文翻译

### 底层改动

- **终端运行时快照与守卫**
  - `TerminalService` 新增 `TerminalRuntimeSnapshot`，包含 `runtimeState`、`reconnectable`、`canRunCommand`、`canWrite` 与 `canUseFilesystem`
  - 共享的 `terminal_runtime_guard` 辅助函数统一了 terminal 解析错误、不可用 terminal 提示，以及终端/文件/等待/读取/传输工具里的状态头
  - 被用户提及的 terminal 上下文现在使用 display terminals，并在最近输出旁注入 terminal status，让 Agent 的上下文与后续工具实际执行约束一致
- **重连管线**
  - `reconnect_terminal_tab` 复用现有 terminal reconnect 路径，并等待最多 45 秒确认 tab ready；成功、再次断开、仍在初始化或 tab 已关闭都会明确返回
  - 断开的 `write_stdin` 会在命令策略评估和 action model 之前被拒绝，因此不可用 terminal 不会触发多余审批或 action-model 调用
- **文件工具能力映射**
  - `write_file`、`edit_file` 与旧 `create_or_edit` 都会映射回单一的 `create_or_edit` 内置能力，用于 Settings、启用状态摘要和禁用工具检查
  - prompt 文案已更新，memory 编辑、skill 编辑和临时代码文件创建都会指向正确的具体工具
- **传输面板模型抽取**
  - 传输任务可见性、终止状态识别、稳定排序、摘要计数，以及当前/后台/最近分组都抽取到 `fileTransferPresentation.ts`
  - 新增展示层测试覆盖当前 tab 传输、后台 Agent 传输、最近终止状态和稳定排序
- **布局持久化行为**
  - `LayoutStore` 会在去抖保存、同步 flush、直接保存和应用已保存布局之前更新活跃 saved-layout snapshot
  - 测试覆盖了动态槽位修改、已保存布局之间切换、已保存布局中的活跃 tab 变化，以及移除手动覆盖流程
- **测试**
  - 新增和扩展的 extreme 规格覆盖终端运行时守卫、重连行为、断开终端工具预检查、拆分后的文件修改工具、传输面板展示模型、SSH keepalive 连接配置、浮层菜单定位、紧凑菜单 portal 拖拽桥接，以及活跃 mention 建议项滚动可见性

### 这次升级带来的实际体验

- 如果 SSH tab 断开，GyShell 更不容易把陈旧终端历史当成实时状态继续操作。
- 远程 SSH terminal 在空闲网络环境下应更稳定。
- Agent 的文件编辑语义更清楚：整文件写入和精确替换已经分开。
- 同时存在多个用户/Agent 文件传输时，进度面板更容易读懂。
- 已保存布局槽位更像会跟随你变化的工作区预设：编辑当前预设，它就会自动更新。
- 紧凑 tab 选择器和 `@` 提及菜单更不容易开到屏幕外，也更不容易遮住当前选中项。
