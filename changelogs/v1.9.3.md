# v1.9.3 Release Notes

## English

v1.9.3 is a connectivity patch. It **expands the SSH `legacy` and `cisco` algorithm presets** so RTerm can connect to older Cisco and embedded network devices that only offer legacy cryptography — devices that previously failed with a `c->s` algorithm-negotiation error before authentication was even attempted.

### The problem

Older Cisco IOS / IOS-XE and embedded switches only offer legacy SSH algorithms. RTerm's ssh2 client, using the default (`modern`) preset or the previous `legacy`/`cisco` presets, didn't offer some of them, so the handshake aborted with a `c->s` error — even though system OpenSSH connected fine.

### The fix

Both presets now offer the **complete set of legacy algorithms ssh2 actually supports at runtime**, while keeping modern/strong entries first so newer images still prefer them:

- **KEX:** added `diffie-hellman-group-exchange-sha1`, `diffie-hellman-group-exchange-sha256`, and `diffie-hellman-group15/16/17/18-sha512` (legacy preset), plus group-exchange variants (cisco preset).
- **Host keys:** added `ecdsa-sha2-nistp384` and `ecdsa-sha2-nistp521` (legacy preset). `ssh-rsa` and `ssh-dss` were already covered.
- **Ciphers:** added `chacha20-poly1305@openssh.com` and the `@openssh.com` GCM variants (legacy preset). CBC/CTR/3des were already covered.
- **HMAC:** added `hmac-ripemd160`, `hmac-sha1-etm@openssh.com`, `hmac-sha2-256-etm@openssh.com`, `hmac-sha2-512-etm@openssh.com`, `hmac-sha2-256-96`, and `hmac-sha2-512-96` (both presets).

### Safety: every offered algorithm is verified ssh2-supported

ssh2 **throws `Unsupported algorithm`** if you offer an algorithm it can't actually use — so offering non-working ones would *crash* connections instead of fixing them. Notably, **RC4 / blowfish / cast ciphers are intentionally excluded**: Node's OpenSSL 3 builds don't provide them (legacy provider), so offering them would break on modern hosts. The expanded presets were validated against ssh2's runtime `SUPPORTED_*` sets so every offered algorithm is one ssh2 will actually negotiate — no crashes, on any host.

### How to use

In **Manage Connection…** for an old device: set **Algorithms preset** to `cisco` (or `legacy` for very old gear) and **Terminal type** to `vt100`.

### Verification

- **18 new tests** (`sshAlgorithmPresets.extreme.spec.ts`) covering: all required legacy/cisco kex, host keys, ciphers, and MACs are present; **every offered algorithm is ssh2-supported** (no `Unsupported algorithm` crash); RC4/blowfish/cast are correctly excluded; and modern-first ordering is preserved.
- `npm test` — **942 tests, 0 failures** (924 + 18 new)
- `npm run typecheck:backend` — clean
- `npm run lint` — 0 errors
