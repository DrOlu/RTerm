# v1.8.1 Release Notes

## English

v1.8.1 is a bug-fix and hardening release. It repairs the developer tooling that was broken in the v1.8.0 tree — the npm `lint` script had no ESLint configuration and two `test:*` scripts pointed at files that no longer exist — and adds a proper GitHub Actions CI workflow that runs the full quality gate (typecheck, lint, and every unit/extreme test suite) on every push and pull request. No user-facing behaviour changes.

### Fixes

- **Fix: `npm run lint` now works.** The v1.8.0 tree shipped an `eslint .` script but no ESLint configuration file, so every lint run failed with *"ESLint couldn't find a configuration file"*. The repo now carries a root `.eslintrc.json` (TypeScript + react-hooks + prettier) tuned for a terminal-emulation codebase: control-character regexes (ANSI escape handling) and lazy `require()` native-module loads are intentionally allowed, while real problems (unused vars, `prefer-const`, useless escapes) still surface as warnings. Also added the missing `eslint-plugin-react-hooks` devDependency that in-source `eslint-disable` comments already referenced.
- **Fix: broken test scripts removed.** `test:backend-regression` and `test:backend-extreme` pointed at `tests/backend-*.cjs` files that do not exist (the `tests/` directory is gitignored and was never committed). They failed every time with `Cannot find module`. They have been replaced by a single top-level `npm test` that runs the four suites that actually exist: layout/UI extreme, backend unit extreme, automation extreme, and the deprecated-CLI guard test.
- **Fix: 9 latent code-quality issues auto-fixed.** `prefer-const` violations in `templateEngine.ts`, `InputParseHelper.ts`, `edit_tools.ts`, and `schedulerService.extreme.spec.ts`, plus a `prefer-as-const` cleanup — all verified with a full typecheck afterwards.

### CI

- **New `test.yml` GitHub Actions workflow.** Every push to `main`/`fix/**`/`feat/**` and every PR now runs, on Ubuntu with Node 20:
  1. `npm run typecheck:all` (node + web + backend + mobile-web)
  2. `npm run lint`
  3. `npm run test:layout-ui-extreme`
  4. `npm run test:backend-unit-extreme`
  5. `npm run test:automation-extreme`
  6. `npm run test:desktop-cli-deprecated`
  7. `npm run build:backend` + `npm run build:electron` smoke build

  Native modules (`better-sqlite3`, `node-pty`) are rebuilt for the Node runtime in CI so the backend suites execute against real binaries.
