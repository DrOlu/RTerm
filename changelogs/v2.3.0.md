# v2.3.0 Release Notes

## English

v2.3.0 adds the **predictive analytics, behavioral analytics, and embedded-evals** layer on top of RTerm's SRE/APM/DEM pillars: statistical anomaly detection, predictive early-warning, UEBA-style behavioral analytics, and an embedded eval harness that measures RTerm's own agent performance, safety, and reliability — all fully unit-tested (28 new tests, 1046 total) and dependency-free (no heavy ML framework; the LLM agent already provides the reasoning layer).

### Embedded evals (`evals/evalHarness.ts`)

An embedded eval harness that measures how well RTerm's **own agent** performs, safely and reliably — running a golden set of cases (offline with a mock agent or online against the real model) and aggregating a reliability report:

- **Task accuracy** — does the agent produce the right answer (expected-substring match).
- **Tool selection** — does it pick the right tool for the job (expected tool-set match).
- **Safety / policy adherence** — does the command policy block dangerous operations (red-team set asserting `deny`).
- **Determinism / replay** — does a recorded golden conversation replay the same ordered tool sequence (regression detection).
- **Aggregate report** — accuracy %, tool-selection %, safety %, replay %, pass/fail, tokens, latency, per-case reasons.

### Predictive analytics (`predictive/`)

- **`anomalyDetector.ts`** — statistical anomaly detection over the metrics time-series: **z-score** (mean ± k·σ) and **robust z-score** (median ± k·MAD/0.6745), dependency-free, with correct zero-variance handling (any deviation from a constant baseline is anomalous) and minimum-baseline safeguards against false positives.
- **`earlyWarningService.ts`** — predictive failure alerting combining **trend forecasting** ("disk will hit 95% in N days" from the MetricsLedger) with **anomaly detection** ("cpu just spiked 4σ above normal"), emitting predictive `EarlyWarning`s and optionally **proposing a MOP change for predictive auto-remediation**.

### Behavioral analytics (`behavior/behaviorLedger.ts`)

UEBA-style behavioral analytics over the agent run + usage data (`agentRunLedger`):
- **Baselines** — runs/day, tokens/run, error rate, and models seen, over a rolling 14-day window (excluding the current day, so today is compared against prior behavior).
- **Deviation detection** — **run-spike** (unusual activity burst), **token-blowout** (outsize run), **error-spike** (abnormal failure rate), and **unusual-model** (a model not in the baseline).
- Ingests `AgentRunRecord`s directly.

All four modules are **pure + injectable** (agent runner, policy checker, and ledgers injected) following RTerm's established pattern, and compose with the existing `metricsLedger`, `agentRunLedger`, `alertService`, `incidentLedger`, and MOP.
