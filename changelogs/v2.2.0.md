# v2.2.0 Release Notes

## English

v2.2.0 completes the SRE work with two integrations: **multi-channel notifications** (Slack, Microsoft Teams, Email/SMTP, Telegram) and a **unified live dashboard** that aggregates every monitoring ledger into one rich, cross-linked view — all fully unit-tested (17 new tests, 1018 total).

### Multi-channel notifications (`notify/notifyService.ts`)

RTerm can now push rich, severity-colored alerts to your team's channels, plugging straight into the v2.0.0 `AlertService` routing engine (grouping, dedupe, silences, severity thresholds):

- **Slack** — incoming-webhook Block Kit with a severity-colored attachment, facts (severity, source, count, detail, labels, time), and the alert fingerprint.
- **Microsoft Teams** — Power Automate / webhook MessageCard with a severity theme color, facts, and source.
- **Email / SMTP** — HTML + plain-text email with a severity-colored bar, facts table, and escaping (nodemailer-compatible sender injected).
- **Telegram** — Markdown chat message with severity emoji, facts, and labels (reuses the proven pattern).

Each channel is a small **factory that produces an `AlertChannel`** for `AlertService.channels`, with an injected HTTP/SMTP sender — fully testable offline, production-ready with vaulted webhook URLs / SMTP creds.

### Unified live dashboard (`dashboard/dashboardService.ts`)

A single `dashboard:state` object that aggregates **every** monitoring ledger into a rich, cross-linked view, broadcast over the gateway for the Dashboard UI and any WebSocket client:

- **Fleet health grid** — per-host golden signals (cpu/mem/disk/traffic) + uptime status.
- **SLO board** — SLI, error budget, burn rate per SLO.
- **Uptime map** — up/degraded/down per host.
- **Incident feed** — open incidents with timelines.
- **APM view** — bottleneck services + slowest traces.
- **DEM view** — slowest/poor pages by Core Web Vitals.
- **k8s/cloud cluster health**.
- **Capacity forecast** — days-to-disk-full per host.

Plus a compact **`summary()`** line for the agent/gateway ("2 hosts · 1 degraded · 1 open incident · SLOs healthy"). All ledgers are injected (pure + deterministic); optional ledgers (APM/DEM/k8s) degrade gracefully when absent.
