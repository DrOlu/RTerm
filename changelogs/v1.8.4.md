# v1.8.4 Release Notes

## English

v1.8.4 is a bug-fix release. It fixes the **standalone `gybackend` headless daemon**, which previously crashed on startup with `ERR_MODULE_NOT_FOUND` and could not be run at all. RTerm can now be run fully headless (`npm run start:backend`) and driven over its WebSocket gateway with no desktop UI.

### The bug

Running the daemon (`node apps/gybackend/dist/index.js`) failed immediately:

```
Error [ERR_MODULE_NOT_FOUND]: Cannot find module
'.../packages/shared/src/terminalConnections' imported from
'.../packages/shared/src/index.ts'
```

Root cause (three parts):
1. **`@gyshell/shared` is a raw TypeScript package, never compiled.** Its `package.json` points `exports` straight at `./src/index.ts` (a symlinked workspace dir), and its build script is a no-op stub.
2. **The daemon bundle marked every package external** (`esbuild --packages=external`), so `@gyshell/shared` was left out of the bundle and resolved at runtime to that raw `index.ts`.
3. **That `index.ts` uses extensionless ESM re-exports** (`export * from './terminalConnections'`). Node's ESM loader does **not** add extensions, so it looked for a file literally named `terminalConnections` (no `.ts`) and threw.

The desktop Electron app was unaffected because electron-vite bundles `@gyshell/shared` and its resolver *does* extension-guess — so the same code ran fine in the app and only broke in the standalone daemon.

### The fix

The daemon build now **bundles `@gyshell/shared`** while keeping real `node_modules` (native deps like `better-sqlite3`, `node-pty`, `ssh2`) external:

```
esbuild src/index.ts --bundle --platform=node --format=esm \
  --packages=external \
  --alias:@gyshell/shared=../../packages/shared/src/index.ts \
  --outfile=dist/index.js
```

esbuild resolves the extensionless imports at build time, so the daemon is self-contained and no longer depends on runtime TS resolution.

### Verification

- `node apps/gybackend/dist/index.js` now **starts cleanly** (`[gybackend] Started.`, listens on its WebSocket port).
- Its gateway answers RPC: `gateway:ping` → `pong`, `terminal:list`, and `terminal:createTab` creates a ready terminal — all confirmed live.
- `npm run typecheck:backend` — clean
- `npm run lint` — 0 errors
- `npm test` — 834 tests, 0 failures

### Note

This is independent of (and in addition to) the v1.8.3 SSH `authMethod` fix. The desktop app's built-in gateway was always unaffected by this daemon build bug.
